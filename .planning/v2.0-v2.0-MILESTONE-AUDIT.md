## Integration Findings

### Missing Connections (Critical)

1. **Storage metrics pipeline**
   - `lsm/compaction_metrics.zig` is not wired into `storage_metrics.update_*` call sites.
   - Dashboard panels depend on `archerdb_compaction_*` metrics that never update.

2. **Query performance metrics export**
   - `QueryMetrics`, `BatchQueryMetrics`, `PreparedQueryMetrics`, `QueryLatencyBreakdown`, and `SpatialIndexStats` are tracked but not exported through `/metrics`.
   - `metrics_server.handleMetrics` only uses `metrics.Registry.format`, which omits these exporters.

3. **RAM index metrics update**
   - `RamIndex.update_prometheus_metrics` has no invocation, leaving `archerdb_ram_index_*` gauges stale.

## Broken E2E Flows

| Flow | Break Point | Impact |
|------|------------|--------|
| Query performance observability | Metrics export to `/metrics` | Cache hit ratio, batch/prepared rates, latency breakdown panels never populate |
| RAM index observability | Metrics update call missing | RAM index dashboards show zeros/stale data |
| Storage optimization observability | Metrics update call missing | Compaction, compression, throttle metrics never reflect runtime behavior |
