  <files>
    tools/s2_golden_gen/main.go
    tools/s2_golden_gen/go.mod
  </files>
  <action>
Create a Go program that generates golden test vectors using Google's S2 library:

1. Create `tools/s2_golden_gen/go.mod`:
   ```
   module archerdb/s2_golden_gen
   go 1.21
   require github.com/golang/geo v0.0.0-20230421003525-6adc56603217
   ```

2. Create `tools/s2_golden_gen/main.go` that generates:
   - **cell_id_golden.tsv**: 1000+ vectors covering:
     - Grid of coordinates (every 10 degrees lat/lon)
     - All 6 faces explicitly tested
     - Poles (lat=89.999999, -89.999999)
     - Antimeridian (lon=179.999999, -179.999999)
     - All S2 levels (0-30) for subset of coordinates
     - Format: `lat_nano\tlon_nano\tlevel\tcell_id_hex`

   - **hierarchy_golden.tsv**: 500+ vectors covering:
     - Parent at each level (30 down to 0)
     - Children for levels 0-29
     - Format: `cell_id_hex\tparent_hex\tchild0_hex\tchild1_hex\tchild2_hex\tchild3_hex`

   - **neighbors_golden.tsv**: 200+ vectors covering:
     - 4 edge neighbors for various cells
     - Format: `cell_id_hex\tn0_hex\tn1_hex\tn2_hex\tn3_hex`

   - **covering_golden.tsv**: 100+ vectors covering:
     - Spherical caps of various radii
     - Format: `lat_nano\tlon_nano\tradius_m\tmin_level\tmax_level\tmax_cells\tcell_ids_comma_sep`

The generator must:
- Use nanodegrees (multiply degrees by 1e9, cast to int64)
- Output cell IDs in hex format (16 chars, zero-padded)
- Include header row with column names
- Generate deterministically (no randomness)
  </action>
