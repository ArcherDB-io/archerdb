<task type="auto">
  <name>Task 2: Add process metrics collection</name>
  <files>src/archerdb/metrics_server.zig</files>
  <action>
Add process-level metrics collection to metrics_server.zig:

1. Create `collectProcessMetrics()` function that reads system info:
   - `process_resident_memory_bytes` - Read from /proc/self/status (VmRSS) on Linux, use mach_task_info on Darwin
   - `process_virtual_memory_bytes` - Read from /proc/self/status (VmSize)
   - `process_cpu_seconds_total` - Read from /proc/self/stat
   - `process_open_fds` - Count entries in /proc/self/fd
   - `process_threads` - Read from /proc/self/status (Threads)

2. Call `collectProcessMetrics()` in handleMetrics() before formatting response.

3. Add `process_start_time_seconds` (Gauge) - capture at server start, never changes.

4. For Darwin (macOS), use alternative syscalls:
   - Use `mach_task_basic_info` for memory
   - Use `getrusage()` for CPU time
   - Skip FD count if /proc not available

Handle platform differences with comptime checks:
```zig
if (@import("builtin").os.tag == .linux) {
    // Linux-specific implementation
} else if (@import("builtin").os.tag == .macos) {
    // Darwin-specific implementation
}
```

These are standard Prometheus process metrics that all exporters provide.
