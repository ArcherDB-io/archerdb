3. Time Series: "Lag by Time" - `archerdb_vsr_replication_lag_seconds` per instance, with 30s warning line, 120s critical line
4. Time Series: "Lag by Operations" - `archerdb_vsr_replication_lag_ops` per instance, with 1000 warning, 10000 critical lines

Row 3 - View History:
5. Time Series: "View Changes" - `increase(archerdb_vsr_view_changes_total[5m])` to show view change frequency
6. Time Series: "Operation Number" - `archerdb_vsr_op_number` per instance to show commit progress

Row 4 - Primary & Replica Status:
7. Stat: "Primary Node" - Use transforms to show which instance has is_primary=1
8. Time Series: "Replica Health" - Derived from lag metrics, show healthy/degraded/unhealthy state

**Panel requirements:**
- Table panel shows instant values, not time series
- Lag panels have threshold lines as annotations
- Description explains VSR concepts: view, primary election, replication lag
- Link to future runbook for "What to do when replication is lagging"

<task type="auto">
  <name>Task 2: Create Storage detail dashboard</name>
  <files>observability/grafana/dashboards/archerdb-storage.json</files>
  <action>
Create the storage detail dashboard for LSM and disk monitoring:

**Dashboard settings:**
- uid: "archerdb-storage"
- title: "ArcherDB Storage"
- tags: ["archerdb", "storage", "lsm", "detail"]
- schemaVersion: 39
- refresh: "30s"

**Template variables:** Same as overview (datasource, instance, terminology)

**Dashboard links:** Back to overview + other detail dashboards

**Panels (2 per row, w=12, h=8):**

Row 1 - LSM Overview:
1. Time Series: "LSM Level Sizes" - `archerdb_lsm_level_size_bytes` by level, stacked area
2. Time Series: "Tables per Level" - `archerdb_lsm_tables_count` by level, stacked bar

Row 2 - Compaction:
3. Time Series: "Compaction Duration" - histogram_quantile p50, p95, p99 of archerdb_compaction_duration_seconds
4. Time Series: "Compaction Throughput" - `rate(archerdb_compaction_bytes_read_total[5m])` and bytes_written

Row 3 - Write Amplification:
5. Time Series: "Write Amplification Ratio" - `archerdb_lsm_write_amplification_ratio` - important for understanding storage efficiency
6. Stat: "Compaction Count" - `increase(archerdb_compaction_total[1h])` compactions per hour

Row 4 - Disk I/O:
7. Time Series: "Disk Read Latency" - histogram_quantile of archerdb_disk_read_latency_seconds
8. Time Series: "Disk Write Latency" - histogram_quantile of archerdb_disk_write_latency_seconds

Row 5 - Memory Breakdown:
9. Time Series: "Memory by Component" - stacked: archerdb_memory_ram_index_bytes, memory_cache_bytes, (used - index - cache) as "other"
10. Time Series: "Data File Size" - `archerdb_data_file_size_bytes` to track storage growth

Row 6 - Checkpoint:
11. Time Series: "Checkpoint Duration" - histogram_quantile of archerdb_checkpoint_duration_seconds
12. Stat: "Checkpoints/hour" - `increase(archerdb_checkpoint_total[1h])`

**Panel requirements:**
- LSM level panels show progression through levels (L0 -> L6)
- Write amplification explained in description (ratio of bytes written to disk vs bytes from client)
- Compaction throughput shows read vs write separately to identify asymmetry
- Memory breakdown helps identify what's using RAM
- Disk latency panels have threshold lines for SSD expectations (<10ms p99)
  </action>
