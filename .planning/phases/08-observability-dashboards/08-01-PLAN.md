<task type="auto">
  <name>Task 2: Create Overview dashboard</name>
  <files>observability/grafana/dashboards/archerdb-overview.json</files>
  <action>
Create the main overview dashboard JSON following RESEARCH.md patterns and CONTEXT.md decisions:

**Dashboard settings:**
- uid: "archerdb-overview"
- title: "ArcherDB Overview"
- tags: ["archerdb", "overview"]
- schemaVersion: 39
- refresh: "30s"
- Theme-agnostic (no hardcoded colors)

**Template variables (in this order):**
1. `datasource` - Prometheus datasource selector (type: datasource)
2. `instance` - Node filter from `label_values(archerdb_info, instance)`, multi-select, include all
3. `terminology` - Custom variable: archerdb|database|plain (per CONTEXT.md)

**Dashboard links (drill-down to detail dashboards):**
- Queries Detail -> /d/archerdb-queries
- Replication Detail -> /d/archerdb-replication
- Storage Detail -> /d/archerdb-storage
- Cluster Detail -> /d/archerdb-cluster

**Annotations:**
- Restarts: `changes(archerdb_vsr_view[5m]) > 0` with orange icon
- Recent starts: `(time() - process_start_time_seconds) < 300` with red icon

**Panels (2 per row, w=12 each, h=8):**

Row 1 - Health Overview:
1. Stat: "Cluster Health" - `min(archerdb_health_status)` (0=healthy, shows worst node)
2. Stat: "Active Nodes" - `count(archerdb_health_status == 0)` / `count(archerdb_health_status)`

Row 2 - Throughput:
3. Time Series: "Write Throughput" - `sum(rate(archerdb_write_operations_total{instance=~"$instance"}[5m])) by (operation)`
4. Time Series: "Read Throughput" - `sum(rate(archerdb_read_operations_total{instance=~"$instance"}[5m])) by (query_type)`

Row 3 - Latency:
5. Time Series: "Write Latency" - histogram_quantile for p50, p95, p99 of archerdb_write_latency_seconds
6. Time Series: "Read Latency" - histogram_quantile for p50, p95, p99 of archerdb_read_latency_seconds

Row 4 - Replication & Resources:
7. Time Series: "Replication Lag" - `max(archerdb_vsr_replication_lag_seconds{instance=~"$instance"})` and ops variant
8. Gauge: "Memory Usage" - `avg(archerdb_memory_used_bytes / archerdb_memory_allocated_bytes) * 100` with 70%/85% thresholds

**Panel requirements:**
- All panels use `$datasource` and filter by `instance=~"$instance"`
- Include description field with what panel shows and why it matters
- Latency panels: yellow threshold at 500ms, red at 2s
- Memory gauge: yellow at 70%, red at 85%
- Use theme-agnostic colors (green/yellow/red from threshold steps)
  </action>
