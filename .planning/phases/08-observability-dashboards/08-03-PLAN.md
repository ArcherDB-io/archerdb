**archerdb-warnings.yaml** (for dashboard indicators, Slack notifications):
```yaml
groups:
  - name: archerdb-latency-warnings
    rules:
      - alert: ArcherDBHighQueryLatencyWarning
        expr: histogram_quantile(0.99, sum(rate(archerdb_read_latency_seconds_bucket[5m])) by (le, instance)) > 0.5
        for: 5m
        labels:
          severity: warning
          team: database
        annotations:
          summary: "High query latency on {{ $labels.instance }}"
          description: "p99 query latency is {{ $value | humanizeDuration }} (threshold: 500ms)"
          runbook_url: "https://docs.archerdb.io/runbooks/high-query-latency"
          remediation: "Check for slow queries, index health, or resource contention"

  - name: archerdb-replication-warnings
    rules:
      - alert: ArcherDBReplicationLagTimeWarning
        expr: max(archerdb_vsr_replication_lag_seconds) by (instance) > 30
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "Replication lag >30s on {{ $labels.instance }}"
          description: "Current lag: {{ $value | humanizeDuration }}"
          runbook_url: "https://docs.archerdb.io/runbooks/replication-lag"
          remediation: "Check network connectivity, replica health, and disk I/O"

      - alert: ArcherDBReplicationLagOpsWarning
        expr: max(archerdb_vsr_replication_lag_ops) by (instance) > 1000
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "Replication lag >1000 ops on {{ $labels.instance }}"
          description: "Current lag: {{ $value }} operations behind"
          runbook_url: "https://docs.archerdb.io/runbooks/replication-lag"
          remediation: "Check network connectivity, replica health, and disk I/O"

  - name: archerdb-resource-warnings
    rules:
      - alert: ArcherDBHighMemoryUsageWarning
        expr: (archerdb_memory_used_bytes / archerdb_memory_allocated_bytes) * 100 > 70
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Memory usage >70% on {{ $labels.instance }}"
          description: "Current usage: {{ $value | humanize }}%"
          runbook_url: "https://docs.archerdb.io/runbooks/memory-pressure"
          remediation: "Check for memory leaks, consider increasing allocation, or reduce workload"

      - alert: ArcherDBHighDiskUsageWarning
        expr: (archerdb_data_file_size_bytes / 1073741824) > 100
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Data file >100GB on {{ $labels.instance }}"
          description: "Current size: {{ $value | humanize }}GB"
          runbook_url: "https://docs.archerdb.io/runbooks/disk-growth"
          remediation: "Check for TTL expiration, compaction effectiveness, or plan storage expansion"

  - name: archerdb-error-warnings
    rules:
      - alert: ArcherDBErrorRateWarning
        expr: sum(rate(archerdb_write_errors_total[5m])) by (instance) > 0.1
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Write errors occurring on {{ $labels.instance }}"
          description: "Error rate: {{ $value | humanize }}/sec"
          runbook_url: "https://docs.archerdb.io/runbooks/write-errors"
          remediation: "Check disk health, available space, and error logs"
```

**archerdb-critical.yaml** (for PagerDuty, immediate response):
```yaml
groups:
  - name: archerdb-latency-critical
    rules:
      - alert: ArcherDBHighQueryLatencyCritical
        expr: histogram_quantile(0.99, sum(rate(archerdb_read_latency_seconds_bucket[5m])) by (le, instance)) > 2
        for: 5m
        labels:
          severity: critical
          team: database
        annotations:
          summary: "Critical query latency on {{ $labels.instance }}"
          description: "p99 query latency is {{ $value | humanizeDuration }} (threshold: 2s)"
          runbook_url: "https://docs.archerdb.io/runbooks/high-query-latency"
          remediation: "Immediate investigation required: check active queries, disk I/O, memory pressure"

  - name: archerdb-replication-critical
    rules:
      - alert: ArcherDBReplicationLagTimeCritical
        expr: max(archerdb_vsr_replication_lag_seconds) by (instance) > 120
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "Critical replication lag >2min on {{ $labels.instance }}"
          description: "Current lag: {{ $value | humanizeDuration }}"
          runbook_url: "https://docs.archerdb.io/runbooks/replication-lag"
          remediation: "Immediate investigation: replica may need resync"

      - alert: ArcherDBReplicationLagOpsCritical
        expr: max(archerdb_vsr_replication_lag_ops) by (instance) > 10000
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "Critical replication lag >10000 ops on {{ $labels.instance }}"
          description: "Current lag: {{ $value }} operations behind"
          runbook_url: "https://docs.archerdb.io/runbooks/replication-lag"
          remediation: "Immediate investigation: replica may need resync"

  - name: archerdb-resource-critical
    rules:
      - alert: ArcherDBHighMemoryUsageCritical
        expr: (archerdb_memory_used_bytes / archerdb_memory_allocated_bytes) * 100 > 85
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "Memory usage >85% on {{ $labels.instance }}"
          description: "Current usage: {{ $value | humanize }}%"
          runbook_url: "https://docs.archerdb.io/runbooks/memory-pressure"
          remediation: "Immediate action: reduce workload or increase memory allocation"

      - alert: ArcherDBNodeDown
        expr: up{job="archerdb"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "ArcherDB node {{ $labels.instance }} is down"
          description: "Node has been unreachable for >1 minute"
          runbook_url: "https://docs.archerdb.io/runbooks/node-down"
          remediation: "Check node health, restart if necessary, investigate root cause"

  - name: archerdb-error-critical
    rules:
      - alert: ArcherDBErrorRateCritical
        expr: sum(rate(archerdb_write_errors_total[5m])) by (instance) > 1
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "High write error rate on {{ $labels.instance }}"
          description: "Error rate: {{ $value | humanize }}/sec"
          runbook_url: "https://docs.archerdb.io/runbooks/write-errors"
          remediation: "Immediate investigation: check disk health and error logs"

  - name: archerdb-cluster-critical
    rules:
      - alert: ArcherDBUnhealthyShards
        expr: (archerdb_coordinator_shards_healthy / archerdb_coordinator_shards_total) < 0.75
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "Less than 75% of shards healthy"
          description: "Healthy: {{ $value | humanize }}%"
          runbook_url: "https://docs.archerdb.io/runbooks/unhealthy-shards"
          remediation: "Check individual shard nodes, network connectivity, and cluster state"
```

# ArcherDB Alertmanager Templates

These templates provide receiver configurations for common notification channels.
Copy the relevant template content into your `alertmanager.yml` file.

## Available Templates

- `slack.tmpl` - Slack webhook notifications
- `pagerduty.tmpl` - PagerDuty incident creation
- `opsgenie.tmpl` - OpsGenie alert integration
- `email.tmpl` - Email notifications (requires SMTP config)
- `webhook.tmpl` - Generic webhook for custom integrations

## Usage

1. Copy the receiver configuration from the relevant template
2. Replace placeholder values (e.g., `<YOUR_SLACK_WEBHOOK_URL>`)
3. Add to your `alertmanager.yml` receivers section
4. Configure routing to direct alerts to appropriate receivers

## Routing Example

```yaml
route:
  receiver: 'archerdb-slack-warnings'
  group_by: ['alertname', 'instance']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h
  routes:
    - match:
        severity: critical
      receiver: 'archerdb-pagerduty-critical'
    - match:
        severity: warning
      receiver: 'archerdb-slack-warnings'
```

## Alert Annotations

All ArcherDB alerts include:
- `summary` - Brief description of the alert
- `description` - Detailed information including current values
- `runbook_url` - Link to troubleshooting documentation
- `remediation` - Quick fix suggestion
