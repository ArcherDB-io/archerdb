**Structure (per RESEARCH.md Pattern 1):**

   - Coordinate encoding (nanodegrees, millimeters, centidegrees)
   - ID types (u128 entity_id, correlation_id, etc.)
   - Field constraints and valid ranges

3. **Operations** (AREF-01)
   For each operation, include:
   - Description
   - Request table (field, type, required, description)
   - Response table
   - Error codes (link to error-codes.md)
   - Example in all 5 SDKs using HTML details/summary tabs

   Operations to document:
   - createBatch / commit (insert/upsert events)
   - queryRadius
   - queryPolygon (including holes)
   - getLatest / getLatestBatch
   - deleteEntities

4. **Request/Response Formats** (AREF-02)
   - Batch semantics (atomic commit, per-event results)
   - Pagination (cursor-based, has_more flag)
   - Result ordering (deterministic)

5. **Error Handling** (AREF-03)
   - Error code categories (link to error-codes.md)
   - Retry semantics summary (link to sdk-retry-semantics.md)
   - Common errors per operation

6. **Rate Limits and Quotas** (AREF-04)
   - Connection limits (pool_size)
   - Batch size limits (10,000 events max)
   - Query result limits (10,000 default, configurable)
   - Note: No server-side rate limiting - rely on client connection limits

7. **Wire Protocol** (AREF-05)
   - Protocol overview (binary protocol over TCP)
   - Message framing basics (length-prefixed)
