  <name>Task 2: Create comprehensive troubleshooting guide</name>
  <files>docs/troubleshooting.md</files>
  <action>
Create new docs/troubleshooting.md covering common issues and resolution steps (OPS-08):

**Structure:**

**1. Introduction**
- How to use this guide
- When to escalate to support
- Log location and format reference

**2. Connection Issues**
- "Connection refused" - server not running, wrong port, firewall
- "Connection timeout" - network issues, DNS, load balancer
- "Cluster ID mismatch" - wrong cluster_id in client config
- "TLS handshake failed" - certificate issues, expiry, hostname mismatch

**3. Performance Issues**
- "High latency (P99 > 100ms)" - disk I/O, compaction, batch size
- "Low throughput" - connection pool too small, batch too small
- "Memory usage high" - RAM index growth, tune capacity
- "Disk usage growing" - TTL not configured, compaction behind

**4. Cluster Issues**
- "Cluster won't form quorum" - network partition, clock skew, data corruption
- "Frequent view changes" - network instability, slow disk
- "Replica falling behind" - slow disk on follower, network congestion
- "Split brain suspected" - not possible with VSR, explain why

**5. Query Issues**
- "Radius query returns no results" - coordinate encoding wrong, empty area
- "Polygon query rejects input" - winding order wrong, self-intersection
- "Results seem incomplete" - pagination needed, hit limit

**6. Replication Issues**
- "S3 upload failing" - credentials, bucket permissions, network
- "Replication lag high" - S3 throttling, network bandwidth
- "Spillover files growing" - S3 outage, check spillover directory

**7. Encryption Issues**
- "Decryption failed" - key rotation incomplete, data corruption
- "Key unavailable" - KMS connectivity, IAM permissions

**8. Diagnostic Commands**
- Health check commands (curl /health/detailed)
- Metrics inspection (curl /metrics | grep specific_metric)
- Log analysis patterns (grep for specific errors)
- Data file verification (archerdb verify)

For each issue, follow format:
- **Symptom**: What the user observes
- **Possible Causes**: List of likely causes
- **Resolution**: Step-by-step fix for each cause
- **Prevention**: How to avoid in future

Cross-reference existing docs (error-codes.md, operations-runbook.md, disaster-recovery.md).
