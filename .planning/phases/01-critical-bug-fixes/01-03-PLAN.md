Fix the TTL cleanup based on investigation:

**Potential root causes (from RESEARCH.md):**

1. **Position initialization bug:**
   - `cleanup_scanner.position` starts at 0
   - If entries are at different positions, scan may miss them
   - Check wraparound logic: `if (start_position >= self.capacity) 0 else start_position`

2. **Empty index at cleanup time:**
   - Check if RAM index is populated when cleanup runs
   - Data might be in LSM tree, not RAM index
   - Log `ram_index.get_stats()` before cleanup

3. **Timing issue:**
   - Cleanup might run before entities are inserted
   - Check consensus timestamp passed to cleanup
   - Verify current_time_ns is after entity expiration time

4. **Different data structure:**
   - Entries might be in a structure that cleanup doesn't scan
   - Verify which index structure holds the TTL entities

**Implementation steps:**

1. Run test script with logging to capture actual behavior
2. Identify which of the above causes applies
