<task type="auto">
  <name>Task 1: Build OnlineReshardingController with dual-write cutover flow</name>
  <files>src/sharding.zig</files>
  <action>
    Add OnlineReshardingController to sharding.zig that orchestrates:
    - ReshardingManager for shard mapping + dual-write state
    - OnlineMigrationWorker for background migration

    Required API (public methods):
    - init(allocator, initial_shards, config, topology_manager?)
    - startOnlineResharding(new_shard_count, total_entities)
    - tickMigration(batch: *MigrationBatch) -> updates worker + metrics
    - maybeCutover() -> performs cutover when worker.isReadyForCutover()
    - cancel(reason) -> rollback + metrics reset

    Metrics updates on start/stop:
    - resharding_mode = 2 (online)
    - resharding_status: preparing -> migrating -> finalizing -> idle
    - resharding_dual_write_enabled (based on manager.isDualWriteRequired)
    - resharding_migration_rate, batches_processed, migration_failures, eta_seconds

    Ensure cancel() clears target ring, resets progress, and disables dual-write.
    Add unit tests in sharding.zig to cover start -> migrate -> cutover -> complete
    and cancel -> rollback transitions with expected metrics values.
  </action>
  <verify>./zig/zig build -j4 -Dconfig=lite test:unit -- --test-filter "Resharding"</verify>
  <done>OnlineReshardingController drives dual-write migration and updates metrics correctly.</done>
