<task type="auto">
  <name>Task 1: Add parallel fan-out execution and policy handling</name>
  <files>src/coordinator.zig</files>
  <action>
    Extend coordinator fan-out support with:
    - FanOutPolicy enum: all, majority, best_effort
    - FanOutShardError struct for per-shard failures
    - FanOutResult additions: shards_failed, partial (bool)

    Implement a new fanOutQuery method that accepts:
    - query_type: QueryType
    - shard_query: fn(ctx: *anyopaque, shard: ShardInfo) anyerror![]GeoEvent
    - ctx: *anyopaque
    - policy_override: ?FanOutPolicy (default by query_type)

    Use std.Thread.Pool (or std.Thread.spawn with a bounded semaphore) to execute
    shard_query concurrently across active shards. Aggregate results under a mutex,
    track shards_succeeded/failed, and enforce policy:
    - all: require shards_succeeded == shards_queried
    - majority: require shards_succeeded > shards_queried / 2
    - best_effort: return partial results and mark partial=true

    Update metrics:
    - coordinator_queries_fanout/inc + coordinator_query_latency observe
    - coordinator_fanout_partial_total when partial=true
    - coordinator_query_errors_unavailable/timeout for failures
  </action>
  <verify>./zig/zig build -j4 -Dconfig=lite test:unit -- --test-filter "Coordinator: fan-out"</verify>
  <done>Coordinator can execute fan-out queries concurrently with policy-based success handling.</done>
