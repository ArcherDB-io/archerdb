  <name>Task 1: Extend VOPR with deterministic replay</name>
  <files>
    src/vopr.zig
    scripts/run_vopr.sh
  </files>
  <action>
    Extend VOPR to support deterministic replay for debugging failures:

    1. **Add replay mode to VOPR** (`src/vopr.zig`):
       - Add `--replay <seed>` flag that replays a specific seed with verbose logging
       - In replay mode: enable full debug logging regardless of build mode
       - In replay mode: print each operation before execution for debugging
       - Store the PRNG state at each major decision point (tick boundaries)
       - Add `--replay-from-tick <N>` to skip to a specific tick and replay from there

    2. **Capture decision history**:
       - Create a simple trace format that records: tick number, operation type, parameters
       - On failure: dump the last 1000 decisions to stderr for reproduction
       - Format should be human-readable (one decision per line)

    3. **Update run_vopr.sh**:
       - Add `--replay <seed>` option that builds in Debug mode and runs replay
       - Add `--dump-on-fail` option that captures decision history on failure
       - Default behavior unchanged (ReleaseSafe for speed)

    4. **Add replay validation**:
       - When replaying: verify that PRNG produces identical decisions
       - Assert if replay diverges from expected decision sequence
       - This catches any non-determinism bugs in the simulator

    Note: VOPR already has seed-based determinism. This extends it with tick-level replay capability for debugging specific failure points.
  </action>
  <verify>
    # Test replay mode works
    ./zig/zig build vopr -Dvopr-state-machine=testing -- --lite --requests-max=50 42
    # Replay same seed should produce identical result
    ./zig/zig build vopr -Dvopr-state-machine=testing -Doptimize=Debug -- --replay 42 --requests-max=50
    # Verify new flags are documented
    ./zig/zig build vopr -Dvopr-state-machine=testing -- --help 2>&1 | grep -i replay
  </verify>
  <done>
    - VOPR supports `--replay <seed>` for deterministic replay with debug logging
    - VOPR supports `--replay-from-tick <N>` to skip to specific tick
    - Decision history captured and dumped on failure
    - run_vopr.sh updated with replay support
    - Replay mode verified to produce identical results for same seed
  </done>
