       ```zig
       pub const enterprise = ConfigCluster{
           // Enterprise: NVMe, 16+ cores, 64GB+ RAM
           .lsm_levels = 7,           // More levels for larger data
           .lsm_growth_factor = 8,    // 8x growth per level
           .lsm_compaction_ops = 64,  // Larger memtable before flush
           .lsm_manifest_compact_extra_blocks = 2,
           .lsm_table_coalescing_threshold_percent = 40,
           .block_size = 512 * KiB,   // Larger blocks for NVMe
           // ... other enterprise-tuned params
       };

       pub const mid_tier = ConfigCluster{
           // Mid-tier: SATA SSD, 8 cores, 32GB RAM
           .lsm_levels = 6,           // Fewer levels
           .lsm_growth_factor = 10,   // Higher growth to reduce levels
           .lsm_compaction_ops = 32,  // Smaller memtable
           .lsm_manifest_compact_extra_blocks = 1,
           .lsm_table_coalescing_threshold_percent = 50,
           .block_size = 256 * KiB,   // Smaller blocks for SATA
           // ... other mid-tier params
       };
       ```

    2. **Tune bloom filter** (`src/constants.zig`):
       - Per CONTEXT.md: 14 bits/key for ~0.1% false positive rate

       - Compaction should have dedicated resources (not background)
       - Tune compaction concurrency (lsm_compaction_iops_read_max/write_max)
       - Consider rate limiting compaction I/O if it impacts foreground ops
       - Add compaction priority configuration if needed

       - Larger memtables reduce flush frequency
       - Appropriate growth factor balances write amp vs read amp
