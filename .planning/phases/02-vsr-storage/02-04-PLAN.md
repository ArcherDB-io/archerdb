  <name>Task 3: Document threat model and security appendix</name>
  <files>
    docs/encryption-security.md
  </files>
  <action>
    Create security documentation for auditors:

    1. **Threat model section**:
       - **In scope** (what encryption protects against):
         - Stolen disk/media (data at rest)
         - Unauthorized file access
         - Database file forensics
         - Backup exposure
       - **Out of scope** (what encryption does NOT protect against):
         - Memory attacks (data in memory is unencrypted)
         - Network attacks (covered by TLS, separate concern)
         - Authorized access abuse
         - Side-channel attacks (timing, etc.)
         - Key compromise (key management is separate)
       - **Assumptions**:
         - KEK is securely stored and access-controlled
         - Key rotation is performed per policy
         - AES-NI is available (or explicit software fallback accepted)

    2. **Algorithm details**:
       - AES-256-GCM: NIST SP 800-38D, 256-bit key, 96-bit IV, 128-bit tag
       - Aegis-256: IETF draft, 256-bit key, 256-bit nonce, 128-bit tag
       - Key derivation: How DEK is generated (CSPRNG)
       - Key wrapping: How DEK is encrypted with KEK
       - IV/Nonce handling: How uniqueness is guaranteed

    3. **Compliance mapping**:
       - FIPS 140-2: AES-GCM only (Aegis not certified)
       - PCI-DSS: Both algorithms meet requirements
       - HIPAA: Both algorithms meet requirements
       - SOC 2: Both algorithms meet requirements

    4. **Key revocation procedure**:
       - Immediate actions: What to do in first hour
       - Re-encryption: How to re-encrypt all data with new key
       - Verification: How to verify old key no longer works
       - Audit trail: What to log for compliance
       - Include exact commands in revocation procedure

    5. **Performance impact**:
       - Encryption overhead: % slowdown for writes
       - Decryption overhead: % slowdown for reads
       - Aegis vs GCM comparison (Aegis typically 2-3x faster)
       - Benchmark methodology (how we measured)

    6. **Audit checklist**:
       - [ ] Key stored in approved key management system
       - [ ] Key rotation performed within policy period
       - [ ] Encryption enabled for all data files
       - [ ] NIST test vectors pass
       - [ ] Software fallback disabled in production
  </action>
