   - Run Java tests: `cd src/clients/java && mvn test`

8. Link to docs/testing/ci-tiers.md for CI information

Create docs/testing/ci-tiers.md:
1. Title: "CI Tier Structure"
2. Overview explaining three-tier approach
3. Smoke tier (<5 min):
   - Triggers: every push
   - Scope: basic connectivity, one test per SDK
   - Purpose: fast feedback, gate PRs
4. PR tier (<15 min):
   - Triggers: pull requests
   - Scope: full SDK test suite, single node
   - Purpose: comprehensive validation before merge
5. Nightly tier (2h):
   - Triggers: 2 AM UTC daily
   - Scope: all SDKs, multi-node topologies (1/3/5 nodes)
   - Purpose: full coverage, edge cases
6. Weekly benchmark tier (3h):
   - Triggers: Sunday 2 AM UTC
   - Scope: full benchmark suite across topologies
   - Purpose: performance regression detection
7. Failure handling:
   - Smoke: blocks merge
   - PR: blocks merge
   - Nightly: alerts, doesn't block
   - Weekly: alerts on regression >10%

   - Read latency: P95 <1ms, P99 <10ms
   - Write latency: P95 <10ms, P99 <50ms
4. Running Benchmarks Locally:
   - Quick run: `python test_infrastructure/benchmarks/cli.py run --topology 3`
   - Full suite: `python test_infrastructure/benchmarks/cli.py run --full-suite`
   - With custom ratio: `--read-write-ratio 0.8`
5. Interpreting Results:
   - Throughput (events/sec)
   - Latency percentiles (P50, P95, P99)
   - Confidence intervals
   - Coefficient of variation
6. Regression Detection:
   - Threshold: 10% degradation
   - Statistical test: Welch's t-test
   - How to compare: `cli.py compare --baseline baseline.json`
7. Historical Tracking:
