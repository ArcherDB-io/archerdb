<task type="auto">
  <name>Task 3: Add KEDA ScaledObject template for HPA based on load (OPS-03)</name>
  <files>
    deploy/helm/archerdb/templates/keda.yaml
    deploy/helm/archerdb/values.yaml
  </files>
  <action>
Create KEDA ScaledObject template for horizontal pod autoscaling based on custom metrics.

**IMPORTANT per RESEARCH.md:** KEDA only makes sense for read replicas. Write quorum is fixed (3 or 5 nodes). This template is opt-in since KEDA operator must be installed in the cluster.

1. **templates/keda.yaml**: KEDA ScaledObject (conditional on autoscaling.enabled)
   - Wrap in `{{- if .Values.autoscaling.enabled }}`
   - apiVersion: keda.sh/v1alpha1
   - kind: ScaledObject
   - metadata:
     - name: {{ include "archerdb.fullname" . }}-autoscaler
     - labels: {{ include "archerdb.labels" . | nindent 4 }}
   - spec:
     - scaleTargetRef:
       - apiVersion: apps/v1
       - kind: StatefulSet
       - name: {{ include "archerdb.fullname" . }}
     - minReplicaCount: {{ .Values.autoscaling.minReplicas | default 3 }}
     - maxReplicaCount: {{ .Values.autoscaling.maxReplicas | default 6 }}
     - pollingInterval: {{ .Values.autoscaling.pollingInterval | default 30 }}
     - cooldownPeriod: {{ .Values.autoscaling.cooldownPeriod | default 300 }}
     - triggers:
       - type: prometheus
         metadata:
           serverAddress: {{ .Values.autoscaling.prometheusAddress | default "http://prometheus:9090" }}
           metricName: archerdb_connections_active
           query: sum(archerdb_connections_active{job="{{ include "archerdb.fullname" . }}"})
           threshold: "{{ .Values.autoscaling.connectionThreshold | default 1000 }}"
     - fallback:
       - failureThreshold: 3
       - replicas: {{ .Values.autoscaling.minReplicas | default 3 }}
   - Add comment at top: "# Requires KEDA operator: https://keda.sh/docs/deploy/"
   - Add comment: "# Only for read replicas - write quorum is fixed at replicaCount"

2. **Update values.yaml** (append to existing):
   ```yaml
   # autoscaling - KEDA-based horizontal pod autoscaling (OPS-03)
   # NOTE: Only for read replicas. Write quorum is fixed at replicaCount.
   # Requires KEDA operator installed in cluster: https://keda.sh/docs/deploy/
   autoscaling:
     enabled: false  # Opt-in, requires KEDA operator
     minReplicas: 3
     maxReplicas: 6
     pollingInterval: 30
     cooldownPeriod: 300
     prometheusAddress: "http://prometheus:9090"
     connectionThreshold: 1000
   ```

3. **Add documentation comment in values.yaml** explaining:
   - KEDA is opt-in (autoscaling.enabled: false by default)
   - KEDA operator must be installed separately
   - Only scales read replicas; write quorum is fixed
   - Connection threshold triggers scale-up at 1000 active connections
  </action>
