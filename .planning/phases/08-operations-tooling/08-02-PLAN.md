Convert existing observability manifests to Helm templates:

1. **templates/servicemonitor.yaml**: Prometheus ServiceMonitor (conditional on metrics.serviceMonitor.enabled)
   - Wrap in `{{- if .Values.metrics.serviceMonitor.enabled }}`
   - Use archerdb.fullname for name
   - Use archerdb.labels for labels
   - Add `release: {{ .Release.Name }}` label for Prometheus operator discovery
   - Configure selector to match service labels using archerdb.selectorLabels
   - Endpoint configuration:
     - port: metrics (9100)
     - path: /metrics
     - interval: {{ .Values.metrics.serviceMonitor.interval | default "15s" }}
     - scrapeTimeout: 10s
     - honorLabels: true
   - namespaceSelector: matchNames with .Release.Namespace

2. **templates/prometheusrule.yaml**: PrometheusRule for alerting (conditional on metrics.alerts.enabled)
   - Wrap in `{{- if .Values.metrics.alerts.enabled }}`
   - Use archerdb.fullname for name
   - Use archerdb.labels for labels
   - Include all critical alerts from existing servicemonitor.yaml:
     - ArcherDBReplicaDown: up{job="archerdb"} == 0 for 1m (critical)
     - ArcherDBViewChangeFrequent: increase(archerdb_view_changes_total[5m]) > 3 for 5m (warning)
     - ArcherDBHighLatency: histogram_quantile(0.99, ...) > 0.1 for 5m (warning)
     - ArcherDBIndexDegraded: archerdb_index_probe_limit_hits_total > 0 for 1m (critical)
     - ArcherDBDiskSpaceLow: disk usage > 85% for 5m (warning)
     - ArcherDBCompactionBacklog: > 1GB for 10m (warning)
   - Add Phase 7 latency alerts (from 07-02):
     - ArcherDBP99LatencyWarning: P99 > 25ms for 5m (warning)
     - ArcherDBP99LatencyCritical: P99 > 100ms for 5m (critical)
   - Add disk fill prediction alert:
     - ArcherDBDiskFillPrediction24h: predict_linear(..., 24*3600) > 0.95 (warning)
     - ArcherDBDiskFillPrediction6h: predict_linear(..., 6*3600) > 0.95 (critical)
   - Use template variables for job name: `job="{{ include "archerdb.fullname" . }}"`
   - Make thresholds configurable via values where sensible

3. **Update values.yaml** (append to existing):
   - metrics.serviceMonitor.enabled: true
   - metrics.serviceMonitor.interval: "15s"
   - metrics.alerts.enabled: true
   - metrics.alerts.latencyP99WarningMs: 25
