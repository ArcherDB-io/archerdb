  <action>
Create Helm chart directory structure and files:

1. **Chart.yaml**: Chart metadata
   - apiVersion: v2
   - name: archerdb
   - version: 0.1.0 (chart version)
   - appVersion: "1.0.0" (ArcherDB version)
   - description: "ArcherDB distributed geospatial database"
   - keywords: [database, geospatial, distributed]
   - maintainers section

2. **values.yaml**: Default values with inline documentation comments
   - replicaCount: 3 (minimum for production)
   - image: repository, tag (default to Chart.appVersion), pullPolicy
   - resources: requests (memory: 2Gi, cpu: 1000m), limits (memory: 8Gi, cpu: 4000m)
   - storage: size (10Gi), storageClass ("" for default)
   - ports: client (3000), metrics (9100)
   - config: clusterId (0), development (true for dev, false for prod)
   - terminationGracePeriodSeconds: 60
   - podAntiAffinity: enabled (true), weight (100)
   - metrics: enabled, serviceMonitor.enabled, serviceMonitor.interval (15s)

3. **templates/_helpers.tpl**: Template helpers
   - archerdb.name: Chart name
   - archerdb.fullname: Release name + chart name
   - archerdb.chart: Chart name + version
   - archerdb.labels: Standard labels (app.kubernetes.io/*)
   - archerdb.selectorLabels: Label selectors
   - archerdb.addresses: Generate comma-separated addresses for replicas

4. **templates/statefulset.yaml**: Convert existing statefulset.yaml to Helm template
   - Use helpers for names, labels, selectors
   - Template replicaCount, image, resources, storage from values
   - Use podManagementPolicy: OrderedReady (required for VSR consensus)
   - Include terminationGracePeriodSeconds from values
   - Add pod anti-affinity from values
   - Use HTTP probes for /health/live and /health/ready (port 9100)
   - Add PVC resource-policy: keep annotation to prevent data loss
   - Template the command to use archerdb.addresses helper

5. **templates/service.yaml**: Both services in one file
   - Headless service for StatefulSet DNS (clusterIP: None)
   - Client-facing ClusterIP service for load balancing
   - Include ports for client and metrics

6. **templates/configmap.yaml**: ConfigMap for cluster configuration
   - Template environment variables from values
   - Include ARCHERDB_DEVELOPMENT flag

7. **templates/NOTES.txt**: Post-install instructions
   - Show how to check cluster health
   - Show how to connect clients
   - Link to documentation
