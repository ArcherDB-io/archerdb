Create DR test automation:

1. **scripts/dr-test.sh**: Bash script for DR validation
   ```bash
   #!/bin/bash
   # ArcherDB Disaster Recovery Test Script
   # Usage: ./scripts/dr-test.sh [--local|--k8s] [--backup-bucket=s3://...]

   set -euo pipefail

   # Test categories:
   # 1. Backup verification - verify existing backup is restorable
   # 2. Single replica failure - stop one replica, verify cluster continues
   # 3. Backup restore - restore from backup to new data file
   # 4. Data integrity - verify restored data matches original
   ```

   Implement test functions:
   - `test_backup_verification()`: Run `archerdb backup verify`
   - `test_single_replica_failure()`: Stop replica, check cluster health, restart
   - `test_backup_restore()`: Restore to temp file, verify with `archerdb verify`
   - `test_data_integrity()`: Compare record counts before/after restore

   Add command-line options:
   - `--local`: Test with local binary (for development)
   - `--k8s`: Test with Kubernetes cluster (kubectl commands)
   - `--backup-bucket`: S3 bucket for backup tests
   - `--cluster-id`: Cluster ID for backup operations
   - `--skip-destructive`: Skip tests that stop replicas
   - `--verbose`: Show detailed output

   Output: JSON summary of test results
   ```json
   {
     "tests_run": 4,
     "tests_passed": 4,
     "tests_failed": 0,
     "details": {...}
   ```
