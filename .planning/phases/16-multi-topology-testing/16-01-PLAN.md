Add the following methods to the ArcherDBCluster class:

1. **stop_node(replica: int, graceful: bool = True) -> None**
   - If graceful=True: Send SIGTERM, wait up to 10s, then SIGKILL if needed
   - If graceful=False: Send SIGKILL immediately
   - Store the process's return code but don't remove from _processes dict (needed for restart)
   - Use proc.terminate() for SIGTERM, proc.kill() for SIGKILL

2. **start_node(replica: int) -> None**
   - Restart a previously stopped node
   - Re-spawn process with same config (don't format - data file already exists)
   - Use same command as initial start
   - Re-create log capture thread for the node

3. **kill_node(replica: int) -> None**
   - Shorthand for stop_node(replica, graceful=False)
   - Simulates crash scenario (SIGKILL)

4. **is_node_running(replica: int) -> bool**
   - Return True if process exists and poll() is None
   - Handle case where replica not in _processes dict

5. **get_leader_replica() -> Optional[int]**
   - Check all running nodes' metrics for archerdb_region_info role="primary"
   - Return replica index (not port) of leader, or None if no leader
   - Timeout of 1 second per node check

6. **get_ports() -> List[int]**
   - Return copy of _ports list (for NetworkPartitioner)

7. **get_metrics_ports() -> List[int]**
   - Return copy of _metrics_ports list

Follow the existing code style in cluster.py. Use subprocess methods (terminate/kill) not os.kill.
