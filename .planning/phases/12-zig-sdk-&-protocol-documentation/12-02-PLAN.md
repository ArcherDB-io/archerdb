    # ArcherDB Protocol Reference

    ## Overview
    ArcherDB uses HTTP/JSON for client communication. This document provides
    complete wire format details for custom client implementers.

    ## Data Types

    ### Coordinate Encoding
    - Latitude: nanodegrees (i64), range -90e9 to +90e9
    - Longitude: nanodegrees (i64), range -180e9 to +180e9
    - Conversion: degrees * 1,000,000,000

    ### ID Encoding
    - Entity IDs: 128-bit unsigned integers
    - JSON representation: string (decimal or UUID format)

    ### Timestamp
    - Unix nanoseconds (u64)

    ## Operations

    For each of the 14 operations, document:

    ### 1. Insert Events (POST /events)
    **Request:**
    {
      "events": [{
        "entity_id": "1001",
        "lat_nano": 37774900000,
        "lon_nano": -122419400000,
        "group_id": 1,
        "ttl_seconds": 86400,
        ...
      }],
      "mode": "insert"
    }

    **Response (success):**
    {
      "results": [{"index": 0, "code": 0}],
      "committed": true
    }

    **Response (validation error):**
    {
      "results": [{"index": 0, "code": 9, "message": "LAT_OUT_OF_RANGE"}]
    }

    ### 2. Upsert Events (POST /events with mode=upsert)
    [Same format as insert, document differences]

    ### 3. Delete Entities (DELETE /entities)
    ### 4. Query by UUID (GET /entity/{id})
    ### 5. Query UUID Batch (POST /entities/batch)
    ### 6. Query Radius (POST /query/radius)
    ### 7. Query Polygon (POST /query/polygon)
    ### 8. Query Latest (POST /query/latest)
    ### 9. Ping (GET /ping)
    ### 10. Status (GET /status)
    ### 11. Get Topology (GET /topology)
    ### 12. Set TTL (POST /ttl/set)
    ### 13. Extend TTL (POST /ttl/extend)
    ### 14. Clear TTL (POST /ttl/clear)

    ## Error Handling

    ### HTTP Status Codes
    - 200: Success
    - 400: Validation error
    - 404: Not found
    - 500: Internal error
    - 503: Cluster unavailable

    ### Error Response Format
    {
      "error": {
        "code": 100,
        "message": "INVALID_COORDINATES",
        "details": "latitude 100.0 out of range [-90, +90]"
      }
    }

    ### Error Code Ranges
    Link to docs/error-codes.md for complete reference.

    ## Pagination

    For query operations:
    - Request: limit (u32), cursor (optional string)
    - Response: has_more (bool), cursor (string if has_more)

    ## Authentication

    **Structure:**

    ```markdown
    # ArcherDB curl Examples

    Complete curl examples for all 14 ArcherDB operations.

    ## Prerequisites

    Start a local ArcherDB server:
    ```bash
    # Single node (development)
    ./zig/zig build run -- --port 3001

    # Or using Docker
    docker run -p 3001:3001 archerdb/archerdb
    ```

    ## Quick Health Check
    ```bash
    curl http://localhost:3001/ping
    # Response: {"pong": true}
    ```

    ## Insert Operations

    ### Insert Single Event
    ```bash
    curl -X POST http://localhost:3001/events -H "Content-Type: application/json" -d '{"events":[{"entity_id":"1001","lat_nano":37774900000,"lon_nano":-122419400000,"group_id":1}]}'
    ```

    ### Insert with All Fields
    ```bash
    curl -X POST http://localhost:3001/events -H "Content-Type: application/json" -d '{"events":[{"entity_id":"2001","lat_nano":37774900000,"lon_nano":-122419400000,"group_id":1,"ttl_seconds":3600,"altitude_mm":100000,"velocity_mms":15000,"accuracy_mm":5000,"heading_cdeg":9000}]}'
    ```

    ### Insert Error: Invalid Latitude
    ```bash
    curl -X POST http://localhost:3001/events -H "Content-Type: application/json" -d '{"events":[{"entity_id":"1001","lat_nano":100000000000,"lon_nano":0}]}'
    # Response: {"results":[{"index":0,"code":9}]}
    ```

    ## Upsert Operations

    ### Upsert Event
    ```bash
    curl -X POST http://localhost:3001/events -H "Content-Type: application/json" -d '{"events":[{"entity_id":"1001","lat_nano":37774900000,"lon_nano":-122419400000}],"mode":"upsert"}'
    ```

    ## Delete Operations

    ### Delete Entity
    ```bash
    curl -X DELETE http://localhost:3001/entities -H "Content-Type: application/json" -d '{"entity_ids":["1001"]}'
    # Response: {"deleted_count":1,"not_found_count":0}
    ```

    ### Delete Non-existent Entity
    ```bash
    curl -X DELETE http://localhost:3001/entities -H "Content-Type: application/json" -d '{"entity_ids":["9999999"]}'
    # Response: {"deleted_count":0,"not_found_count":1}
    ```

    ## Query Operations

    ### Get Latest by UUID
    ```bash
    curl http://localhost:3001/entity/1001
    ```

    ### Query UUID Batch
    ```bash
    curl -X POST http://localhost:3001/entities/batch -H "Content-Type: application/json" -d '{"entity_ids":["1001","1002","1003"]}'
    ```

    ### Query Radius
    ```bash
    curl -X POST http://localhost:3001/query/radius -H "Content-Type: application/json" -d '{"center_lat":37.7749,"center_lon":-122.4194,"radius_m":1000,"limit":100}'
    ```

    ### Query Radius with Group Filter
    ```bash
    curl -X POST http://localhost:3001/query/radius -H "Content-Type: application/json" -d '{"center_lat":37.7749,"center_lon":-122.4194,"radius_m":5000,"group_id":1,"limit":100}'
    ```

    ### Query Polygon
    ```bash
    curl -X POST http://localhost:3001/query/polygon -H "Content-Type: application/json" -d '{"vertices":[{"lat":37.79,"lon":-122.42},{"lat":37.79,"lon":-122.39},{"lat":37.76,"lon":-122.39},{"lat":37.76,"lon":-122.42}],"limit":100}'
    ```

    ### Query Latest
    ```bash
    curl -X POST http://localhost:3001/query/latest -H "Content-Type: application/json" -d '{"limit":100}'
    ```

    ## Server Operations

    ### Ping
    ```bash
    curl http://localhost:3001/ping
    # Response: {"pong":true}
    ```

    ### Status
    ```bash
    curl http://localhost:3001/status
    # Response: {"events_count":1234,"index_bytes":56789,...}
    ```

    ### Topology
    ```bash
    curl http://localhost:3001/topology
    # Response: {"version":1,"num_shards":1,"shards":[...],...}
    ```

    ## TTL Operations

    ### Set TTL
    ```bash
    curl -X POST http://localhost:3001/ttl/set -H "Content-Type: application/json" -d '{"entity_id":"1001","ttl_seconds":86400}'
    ```

    ### Extend TTL
    ```bash
    curl -X POST http://localhost:3001/ttl/extend -H "Content-Type: application/json" -d '{"entity_id":"1001","extend_by_seconds":3600}'
    ```

    ### Clear TTL
    ```bash
    curl -X POST http://localhost:3001/ttl/clear -H "Content-Type: application/json" -d '{"entity_id":"1001"}'
    ```

    ## Pagination Example

    ```bash
    # First page
    curl -X POST http://localhost:3001/query/radius -H "Content-Type: application/json" -d '{"center_lat":37.7749,"center_lon":-122.4194,"radius_m":10000,"limit":100}'
    # Response includes: "has_more":true,"cursor":"abc123..."

    # Next page
    curl -X POST http://localhost:3001/query/radius -H "Content-Type: application/json" -d '{"center_lat":37.7749,"center_lon":-122.4194,"radius_m":10000,"limit":100,"cursor":"abc123..."}'
    ```

    ## Common Errors

    | Error | Example | Fix |
    |-------|---------|-----|
    | Invalid latitude | lat_nano > 90e9 | Use range [-90e9, +90e9] |
    | Invalid longitude | lon_nano > 180e9 | Use range [-180e9, +180e9] |
    | Batch too large | >10000 events | Split into smaller batches |
    | Entity not found | Query missing UUID | Check entity_id exists |

    ## See Also

    - [API Reference](api-reference.md) - Detailed operation documentation
    - [Protocol Reference](protocol.md) - Wire format details
    - [Error Codes](error-codes.md) - Complete error reference
