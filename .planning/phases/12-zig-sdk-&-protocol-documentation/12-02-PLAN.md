    ```markdown
    # ArcherDB Protocol Reference

    ## Overview
    ArcherDB uses HTTP/JSON for client communication. This document provides
    complete wire format details for custom client implementers.

    ## Data Types

    ### Coordinate Encoding
    - Latitude: nanodegrees (i64), range -90e9 to +90e9
    - Longitude: nanodegrees (i64), range -180e9 to +180e9
    - Conversion: degrees * 1,000,000,000

    ### ID Encoding
    - Entity IDs: 128-bit unsigned integers
    - JSON representation: string (decimal or UUID format)

    ### Timestamp
    - Unix nanoseconds (u64)

    ## Operations

    ### 1. Insert Events (POST /events)

    **Request:**
    ```json
    {
      "events": [{
        "entity_id": "1001",
        "lat_nano": 37774900000,
        "lon_nano": -122419400000,
        "group_id": 1,
        "ttl_seconds": 86400,
        "altitude_mm": 100000,
        "velocity_mms": 15000,
        "accuracy_mm": 5000,
        "heading_cdeg": 9000
      }],
      "mode": "insert"
    }
    ```

    **Response (success):**
    ```json
    {
      "results": [{"index": 0, "code": 0}],
      "committed": true
    }
    ```

    **Response (validation error):**
    ```json
    {
      "results": [{"index": 0, "code": 9, "message": "LAT_OUT_OF_RANGE"}]
    }
    ```

    ### 2. Upsert Events (POST /events with mode=upsert)

    **Request:**
    ```json
    {
      "events": [{
        "entity_id": "1001",
        "lat_nano": 37774900000,
        "lon_nano": -122419400000,
        "group_id": 1
      }],
      "mode": "upsert"
    }
    ```

    **Response (success):**
    ```json
    {
      "results": [{"index": 0, "code": 0, "updated": true}],
      "committed": true
    }
    ```

    **Response (created new):**
    ```json
    {
      "results": [{"index": 0, "code": 0, "updated": false}],
      "committed": true
    }
    ```

    ### 3. Delete Entities (DELETE /entities)

    **Request:**
    ```json
    {
      "entity_ids": ["1001", "1002", "1003"]
    }
    ```

    **Response (success):**
    ```json
    {
      "deleted_count": 2,
      "not_found_count": 1
    }
    ```

    **Response (empty request):**
    ```json
    {
      "error": {
        "code": 101,
        "message": "EMPTY_REQUEST",
        "details": "entity_ids array is empty"
      }
    }
    ```

    ### 4. Query by UUID (GET /entity/{id})

    **Request:**
    GET /entity/1001

    **Response (found):**
    ```json
    {
      "event": {
        "entity_id": "1001",
        "lat_nano": 37774900000,
        "lon_nano": -122419400000,
        "group_id": 1,
        "timestamp_ns": 1706745600000000000,
        "ttl_seconds": 86400,
        "altitude_mm": 100000,
        "velocity_mms": 15000
      }
    }
    ```

    **Response (not found):**
    ```json
    {
      "event": null
    }
    ```

    ### 5. Query UUID Batch (POST /entities/batch)

    **Request:**
    ```json
    {
      "entity_ids": ["1001", "1002", "1003"]
    }
    ```

    **Response (partial match):**
    ```json
    {
      "events": [
        {"entity_id": "1001", "lat_nano": 37774900000, "lon_nano": -122419400000, ...},
        null,
        {"entity_id": "1003", "lat_nano": 40712800000, "lon_nano": -74006000000, ...}
      ]
    }
    ```

    **Response (batch too large):**
    ```json
    {
      "error": {
        "code": 102,
        "message": "BATCH_TOO_LARGE",
        "details": "Maximum batch size is 1000"
      }
    }
    ```

    ### 6. Query Radius (POST /query/radius)

    **Request:**
    ```json
    {
      "center_lat_nano": 37774900000,
      "center_lon_nano": -122419400000,
      "radius_mm": 1000000,
      "limit": 100,
      "group_id": 1,
      "cursor": null
    }
    ```

    Note: radius_mm is radius in millimeters. 1000000 mm = 1 km.

    **Response (with results):**
    ```json
    {
      "events": [
        {"entity_id": "1001", "lat_nano": 37774500000, "lon_nano": -122419000000, "distance_mm": 50000, ...},
        {"entity_id": "1002", "lat_nano": 37775000000, "lon_nano": -122418000000, "distance_mm": 150000, ...}
      ],
      "has_more": true,
      "cursor": "eyJsYXN0X2lkIjogMTAwMn0="
    }
    ```

    **Response (empty):**
    ```json
    {
      "events": [],
      "has_more": false,
      "cursor": null
    }
    ```

    **Response (invalid radius):**
    ```json
    {
      "error": {
        "code": 103,
        "message": "INVALID_RADIUS",
        "details": "radius_mm must be positive"
      }
    }
    ```

    ### 7. Query Polygon (POST /query/polygon)

    **Request:**
    ```json
    {
      "vertices": [
        {"lat_nano": 37790000000, "lon_nano": -122420000000},
        {"lat_nano": 37790000000, "lon_nano": -122390000000},
        {"lat_nano": 37760000000, "lon_nano": -122390000000},
        {"lat_nano": 37760000000, "lon_nano": -122420000000}
      ],
      "limit": 100,
      "group_id": null,
      "cursor": null
    }
    ```

    Note: Polygon must be closed (first and last vertex should match) or system auto-closes.
    Vertices must be in counter-clockwise order.

    **Response (with results):**
    ```json
    {
      "events": [
        {"entity_id": "1001", "lat_nano": 37774900000, "lon_nano": -122419400000, ...}
      ],
      "has_more": false,
      "cursor": null
    }
    ```

    **Response (invalid polygon):**
    ```json
    {
      "error": {
        "code": 104,
        "message": "INVALID_POLYGON",
        "details": "Polygon must have at least 3 vertices"
      }
    }
    ```

    ### 8. Query Latest (POST /query/latest)

    **Request:**
    ```json
    {
      "limit": 100,
      "group_id": 1,
      "since_ns": 1706745600000000000,
      "cursor": null
    }
    ```

    **Response (with results):**
    ```json
    {
      "events": [
        {"entity_id": "1005", "lat_nano": 37774900000, "lon_nano": -122419400000, "timestamp_ns": 1706745700000000000, ...},
        {"entity_id": "1004", "lat_nano": 40712800000, "lon_nano": -74006000000, "timestamp_ns": 1706745650000000000, ...}
      ],
      "has_more": true,
      "cursor": "eyJ0cyI6IDE3MDY3NDU2NTAwMDAwMDAwMDB9"
    }
    ```

    **Response (empty):**
    ```json
    {
      "events": [],
      "has_more": false,
      "cursor": null
    }
    ```

    ### 9. Ping (GET /ping)

    **Request:**
    GET /ping

    **Response (healthy):**
    ```json
    {
      "pong": true
    }
    ```

    **Response (unhealthy):**
    HTTP 503 Service Unavailable
    ```json
    {
      "pong": false,
      "reason": "cluster not ready"
    }
    ```

    ### 10. Status (GET /status)

    **Request:**
    GET /status

    **Response:**
    ```json
    {
      "events_count": 125000,
      "entities_count": 42000,
      "index_bytes": 56789012,
      "uptime_seconds": 86400,
      "version": "1.0.0",
      "cluster_state": "healthy",
      "node_id": "node-1"
    }
    ```

    ### 11. Get Topology (GET /topology)

    **Request:**
    GET /topology

    **Response (single node):**
    ```json
    {
      "version": 1,
      "num_shards": 1,
      "replication_factor": 1,
      "shards": [
        {
          "shard_id": 0,
          "primary": {"node_id": "node-1", "address": "127.0.0.1:3001"},
          "replicas": []
        }
      ]
    }
    ```

    **Response (clustered):**
    ```json
    {
      "version": 3,
      "num_shards": 4,
      "replication_factor": 3,
      "shards": [
        {
          "shard_id": 0,
          "primary": {"node_id": "node-1", "address": "10.0.0.1:3001"},
          "replicas": [
            {"node_id": "node-2", "address": "10.0.0.2:3001"},
            {"node_id": "node-3", "address": "10.0.0.3:3001"}
          ]
        },
        ...
      ]
    }
    ```

    ### 12. Set TTL (POST /ttl/set)

    **Request:**
    ```json
    {
      "entity_id": "1001",
      "ttl_seconds": 86400
    }
    ```

    **Response (success):**
    ```json
    {
      "success": true,
      "expires_at_ns": 1706832000000000000
    }
    ```

    **Response (entity not found):**
    ```json
    {
      "error": {
        "code": 105,
        "message": "ENTITY_NOT_FOUND",
        "details": "Entity 1001 does not exist"
      }
    }
    ```

    ### 13. Extend TTL (POST /ttl/extend)

    **Request:**
    ```json
    {
      "entity_id": "1001",
      "extend_by_seconds": 3600
    }
    ```

    **Response (success):**
    ```json
    {
      "success": true,
      "previous_expires_at_ns": 1706832000000000000,
      "new_expires_at_ns": 1706835600000000000
    }
    ```

    **Response (no existing TTL):**
    ```json
    {
      "error": {
        "code": 106,
        "message": "NO_TTL_SET",
        "details": "Entity 1001 has no TTL to extend"
      }
    }
    ```

    ### 14. Clear TTL (POST /ttl/clear)

    **Request:**
    ```json
    {
      "entity_id": "1001"
    }
    ```

    **Response (success):**
    ```json
    {
      "success": true,
      "previous_expires_at_ns": 1706832000000000000
    }
    ```

    **Response (no existing TTL):**
    ```json
    {
      "success": true,
      "previous_expires_at_ns": null
    }
    ```

    ## Error Handling

    ### HTTP Status Codes
    - 200: Success
    - 400: Validation error (bad request format, invalid coordinates)
    - 404: Not found (entity lookup only)
    - 500: Internal error
    - 503: Cluster unavailable

    ### Error Response Format
    ```json
    {
      "error": {
        "code": 100,
        "message": "INVALID_COORDINATES",
        "details": "latitude 100.0 out of range [-90, +90]"
      }
    }
    ```

    ### Error Code Ranges
    Link to docs/error-codes.md for complete reference.

    - 1-99: Client errors (validation)
    - 100-199: Request errors
    - 200-299: Server errors
    - 300-399: Cluster errors

    ## Pagination

    For query operations:
    - Request: limit (u32), cursor (optional string)
    - Response: has_more (bool), cursor (string if has_more)

    Example flow:
    1. First request: `{"limit": 100, "cursor": null}`
    2. Response: `{"events": [...], "has_more": true, "cursor": "abc123"}`
    3. Next request: `{"limit": 100, "cursor": "abc123"}`
    4. Continue until `has_more: false`

    ## Authentication

    Authentication is not currently required. All endpoints are open.
    Future versions may add API key or JWT authentication.

    ## See Also

    - [API Reference](api-reference.md) - Operation details
    - [Error Codes](error-codes.md) - Complete error reference

    # ArcherDB curl Examples

    Complete curl examples for all 14 ArcherDB operations.

    ## Prerequisites

    Start a local ArcherDB server:
    ```bash
    # Single node (development)
    ./zig/zig build run -- --port 3001

    # Or using Docker
    docker run -p 3001:3001 archerdb/archerdb
    ```

    ## Quick Health Check
    ```bash
    curl http://localhost:3001/ping
    # Response: {"pong": true}
    ```

    ## Insert Operations

    ### Insert Single Event
    ```bash
    curl -X POST http://localhost:3001/events -H "Content-Type: application/json" -d '{"events":[{"entity_id":"1001","lat_nano":37774900000,"lon_nano":-122419400000,"group_id":1}]}'
    ```

    ### Insert with All Fields
    ```bash
    curl -X POST http://localhost:3001/events -H "Content-Type: application/json" -d '{"events":[{"entity_id":"2001","lat_nano":37774900000,"lon_nano":-122419400000,"group_id":1,"ttl_seconds":3600,"altitude_mm":100000,"velocity_mms":15000,"accuracy_mm":5000,"heading_cdeg":9000}]}'
    ```

    ### Insert Error: Invalid Latitude
    ```bash
    curl -X POST http://localhost:3001/events -H "Content-Type: application/json" -d '{"events":[{"entity_id":"1001","lat_nano":100000000000,"lon_nano":0}]}'
