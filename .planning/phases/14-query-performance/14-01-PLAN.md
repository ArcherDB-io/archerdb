  <name>Task 1: Create QueryResultCache module</name>
  <files>src/query_cache.zig</files>
  <action>
Create `src/query_cache.zig` implementing the query result cache:

1. Define CachedResult struct:
   - query_hash: u64 (key)
   - generation: u64 (for invalidation check)
   - result_len: u32 (length of cached response)
   - result_data: [max_cached_result_size]u8 (serialized response bytes)
   - Use max_cached_result_size = 4096 bytes (fits most dashboard query responses)

2. Define QueryResultCache struct using SetAssociativeCacheType pattern:
   - Key: u64 (query hash)
   - Value: CachedResult
   - Use CLOCK eviction from SetAssociativeCacheType
   - generation: u64 field (incremented on every write)
   - Default cache size: 1024 entries (configurable via value_count_max)

3. Implement cache operations:
   - get(query_hash: u64) -> ?*CachedResult - returns null if miss OR if generation mismatch
   - put(query_hash: u64, result: []const u8) -> void - stores with current generation
   - invalidateAll() -> void - increments generation (wrapping)

4. Implement query hash function:
   - hashQuery(operation: u8, filter_bytes: []const u8) -> u64
   - Use stdx.hash_inline for deterministic cross-platform hashing
   - Include operation type and all filter bytes in hash

5. Add init/deinit functions following SetAssociativeCache pattern

Follow SetAssociativeCacheType layout from src/lsm/set_associative_cache.zig.
Use Layout with ways=16, tag_bits=8, clock_bits=2 for optimal dashboard workload.

Integrate QueryResultCache into GeoStateMachine:

1. Add query_cache field to GeoStateMachineType:
   - query_cache: QueryResultCache (import from query_cache.zig)
   - Initialize in init() function
   - Deinitialize in deinit() function

2. Add cache lookup in query commit paths:
   - In commit_query_uuid: check cache before RAM index lookup
   - In commit_query_radius: check cache before S2 covering computation
   - In commit_query_polygon: check cache before polygon covering computation

   Pattern for each query type:
   ```zig
   const query_hash = QueryResultCache.hashQuery(operation, filter_bytes);
   if (self.query_cache.get(query_hash)) |cached| {
       @memcpy(output[0..cached.result_len], cached.result_data[0..cached.result_len]);
       return cached.result_len;
   }
   // ... existing query execution ...
   // After computing result:
   self.query_cache.put(query_hash, result_bytes);
   ```

3. Add cache invalidation in write commit paths:
   - In commit_insert_geo_events: call query_cache.invalidateAll()
   - In commit_delete_entities: call query_cache.invalidateAll()
   - Write-invalidation ensures freshness without tracking spatial overlap

4. Add cache metrics recording:
   - Add archerdb_query_cache_hits_total Counter to QueryMetrics
   - Add archerdb_query_cache_misses_total Counter to QueryMetrics
   - Record hit on cache get success, miss on get failure
   - Export in QueryMetrics.toPrometheus()
