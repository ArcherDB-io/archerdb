  <action>
Create Python package in `test-infrastructure/harness/` that wraps ArcherDB process lifecycle.

**port_allocator.py:**
- `find_available_port(start=3100, end=4100) -> int`: Find single available port using socket binding
- `allocate_ports(count: int, base_port: int = 0) -> List[int]`: Allocate multiple ports for cluster nodes
- Use `socket.SO_REUSEADDR` to handle quick port recycling
- Never hardcode ports - always dynamically allocate for parallel test safety

**log_capture.py:**
- `LogCapture` class that wraps subprocess stdout/stderr
- `get_logs(max_lines: int = 1000) -> str`: Return captured logs
- `rotate_logs(max_bytes: int = 1048576)`: Rotate when exceeding 1MB
- Thread-safe implementation using `threading.Lock`

**cluster.py:**
- `ClusterConfig` dataclass with fields:
  - `node_count: int = 1`
  - `base_port: int = 0` (0 = auto-allocate)
  - `data_dir: Optional[str] = None` (None = temp dir via tempfile.mkdtemp)
  - `cluster_id: int = 0`
  - `cache_grid: str = "512MiB"` (small for testing)
  - `preserve_on_failure: bool = False` (controlled by PRESERVE_ON_FAILURE env var)
  - `archerdb_bin: str = ""` (auto-detect from PROJECT_ROOT/zig-out/bin/archerdb)
  - `startup_timeout: float = 60.0`
  - `metrics_base_port: int = 0` (0 = auto-allocate, offset from node port)

- `ArcherDBCluster` class with methods:
  - `__init__(config: ClusterConfig)`: Initialize cluster state
  - `start() -> None`: Format data files, start all replicas, wait for ready
  - `stop() -> None`: Gracefully terminate all processes, cleanup unless preserve_on_failure
  - `wait_for_ready(timeout: float = 60) -> bool`: Poll health endpoints until all nodes ready
  - `wait_for_leader(timeout: float = 60) -> Optional[int]`: Wait for leader election, return leader port
  - `get_addresses() -> str`: Return comma-separated "host:port,host:port" for SDK connection
  - `get_leader_address() -> Optional[str]`: Return leader's address for write operations
  - `get_logs(replica: int = 0) -> str`: Return logs for debugging
  - `__enter__` / `__exit__`: Context manager support for `with ArcherDBCluster(config) as cluster:`

- Implementation details:
  - Use `subprocess.Popen` with `stdout=PIPE, stderr=STDOUT` to capture logs
  - Health check via GET to `http://127.0.0.1:{metrics_port}/health/ready`
  - Leader detection via GET to `http://127.0.0.1:{metrics_port}/metrics` and parse `archerdb_cluster_role{role="leader"} 1`
  - Generate addresses string matching dev-cluster.sh format: "3001,3002,3003"
  - Format command: `archerdb format --cluster={id} --replica={i} --replica-count={n} {datafile}`
  - Start command: `archerdb start --addresses={addrs} --cache-grid={size} --metrics-port={port} --metrics-bind=127.0.0.1 {datafile}`
  - Check `PRESERVE_ON_FAILURE` env var to override cleanup behavior

**cli.py:**
- Argparse-based CLI wrapper exposing harness functionality
- Commands: `start`, `stop`, `status`, `logs`
- Options: `--nodes`, `--data-dir`, `--base-port`, `--timeout`
- Example: `python -m test-infrastructure.harness.cli start --nodes=3`

**__init__.py:**
- Export: `ArcherDBCluster`, `ClusterConfig`, `find_available_port`, `allocate_ports`

**requirements.txt:**
- `requests>=2.28.0` (for health check HTTP calls)
- No external dependencies for port allocation (use stdlib socket)

**distributions.py:**
- `uniform_distribution(count: int, lat_range: tuple, lon_range: tuple, rng: random.Random) -> List[tuple]`
  - Generate uniformly distributed lat/lon pairs within bounds
  - Return list of (lat, lon) tuples

- `gaussian_cluster(center_lat: float, center_lon: float, count: int, std_km: float, rng: random.Random) -> List[tuple]`
  - Generate points around center with Gaussian distribution
  - std_km controls spread (e.g., 10km = city center, 50km = metro area)
  - Handle lat/lon wraparound for edge cases

- `city_concentrated(cities: List[str], count: int, concentration: float, rng: random.Random) -> List[tuple]`
  - concentration=0.8 means 80% in city centers, 20% scattered
  - Distribute events across selected cities proportionally
  - Use gaussian_cluster for city events, uniform for scattered

- `hotspot_pattern(hotspot_coords: List[tuple], count: int, hotspot_ratio: float, rng: random.Random) -> List[tuple]`
  - hotspot_ratio=0.95 means 95% at hotspots (stress test for worst case)
  - For testing extreme concentration scenarios
