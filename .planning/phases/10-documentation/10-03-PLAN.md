Create docs/runbooks/ directory and 7 runbook pages following the structure from RESEARCH.md:

1. **replica-down.md** (covers ArcherDBReplicaDown)
   - Severity: critical
   - What it means: A replica is not responding to health checks
   - Immediate actions: Check pod status, node health, network
   - Investigation: kubectl logs, describe pod, events
   - Resolution: Restart pod, check storage, verify network
   - Prevention: PDB configuration, resource limits

2. **view-changes.md** (covers ArcherDBViewChangeFrequent)
   - Severity: warning
   - What it means: Too many leader elections indicating instability
   - Immediate actions: Identify which replicas are failing
   - Investigation: Check network latency, CPU usage, disk I/O
   - Resolution: Fix network issues, increase election timeout
   - Prevention: Stable network, adequate resources

3. **index-degraded.md** (covers ArcherDBIndexDegraded)
   - Severity: warning
   - What it means: Index is operating in degraded mode (hash collisions)
   - Immediate actions: Check memory usage, index size
   - Investigation: Review ram_index_capacity configuration
   - Resolution: Increase ram_index_capacity (Phase 5 optimized to 500K)
   - Prevention: Capacity planning based on entity count

4. **high-read-latency.md** (covers ArcherDBReadLatencyP99Warning, ArcherDBReadLatencyP99Critical)
   - Severity: warning (>25ms), critical (>100ms)
   - What it means: Read queries taking longer than thresholds
   - Immediate actions: Check for compaction activity, disk I/O
   - Investigation: Review query patterns, cache hit rates
   - Resolution: Add indexes, increase cache, reduce query scope
   - Prevention: Query optimization, capacity planning

5. **high-write-latency.md** (covers ArcherDBWriteLatencyP99Warning, ArcherDBWriteLatencyP99Critical, ArcherDBHighLatency)
   - Severity: warning (>25ms), critical (>100ms)
   - What it means: Write operations taking longer than thresholds
   - Immediate actions: Check compaction backlog, WAL usage
   - Investigation: Review write patterns, batch sizes
   - Resolution: Reduce batch size, increase compaction resources
   - Prevention: Write batching optimization

6. **disk-capacity.md** (covers ArcherDBDiskSpaceWarning, ArcherDBDiskSpaceCritical, ArcherDBDiskFillPrediction24h, ArcherDBDiskFillPrediction6h)
   - Severity: warning (>80% or fill in 24h), critical (>90% or fill in 6h)
   - What it means: Disk space running low or trending toward exhaustion
   - Immediate actions: Free space, enable TTL cleanup
   - Investigation: Identify large files, check compaction
   - Resolution: Expand storage, archive old data, increase TTL cleanup
   - Prevention: Capacity monitoring, retention policies

7. **compaction-backlog.md** (covers ArcherDBCompactionBacklog)
   - Severity: warning
   - What it means: LSM compaction falling behind write rate
   - Immediate actions: Check CPU usage, I/O throughput
   - Investigation: Review compaction threads, L0 trigger
   - Resolution: Increase compaction threads, adjust triggers (Phase 5: L0=8, threads=3)
   - Prevention: Tune compaction based on workload
