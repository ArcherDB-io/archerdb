Create security best practices guide for local-only deployment model:

1. Add deployment context section:
   ```markdown
   ## Deployment Model

   ArcherDB is designed for local-only deployment where security is handled
   at the infrastructure level. This guide documents best practices for
   securing your ArcherDB deployment.
   ```

2. Document network security:
   - Firewall configuration (block external access to ports 3000-3002)
   - VPC/private network deployment
   - No public internet exposure
   - SSH tunneling for remote access

3. Document disk security:
   - Full disk encryption (LUKS, FileVault, BitLocker)
   - File permissions (600 for data files)
   - Backup encryption (at rest)
   - Secure deletion procedures

4. Document existing security capabilities (from Phase 6 inventory):
   - TLS support (available but not enabled in local deployment)
   - Encryption at rest (configurable via --encryption flag)
   - Audit logging (available via --audit-log flag)
   - Note: "Enable these features if your threat model requires them"

5. Add operational security:
   - Access control to servers (SSH keys, jump boxes)
   - Log monitoring for unauthorized access
   - Regular security updates
   - Backup verification

6. Add "Security Checklist" at top:
   ```markdown
   ## Quick Security Checklist

   - [ ] ArcherDB ports (3000-3002) not exposed to internet
   - [ ] Data files have restricted permissions (600)
   - [ ] Full disk encryption enabled
   - [ ] Regular backups with encryption
   - [ ] SSH access restricted to authorized users
   ```

7. Cross-link to:
   - disaster-recovery.md (backup procedures)
   - operations-runbook.md (access management)
