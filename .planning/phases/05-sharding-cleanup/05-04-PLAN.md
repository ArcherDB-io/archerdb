<task type="auto">
  <name>Task 2: Integrate tiering with RAM index and query routing</name>
  <files>src/lsm/groove.zig, src/lsm/forest.zig, src/ram_index.zig</files>
  <action>
Connect tiering decisions to RAM index management and query routing:

1. **RAM index tier awareness:**
   - Modify RAM index to check tier before adding entity
   - Hot and warm tier entities go in RAM index
   - Cold tier entities are NOT in RAM index (disk-only)
   - On demotion to cold: remove from RAM index
   - On promotion from cold: add to RAM index

2. **Query routing based on tier:**
   - For uuid queries: check isInRamIndex() first
   - If not in RAM index, fall back to LSM scan (cold tier query)
   - Track cold tier queries in tiering.stats.cold_tier_queries

3. **Groove integration:**
   - Pass tiering reference to Groove
   - Use tier information for prefetch decisions
   - Prioritize hot tier data in cache

4. **Forest integration:**
   - Access tiering during tree operations
   - Consider tier when deciding eviction candidates

5. **Metrics exposure:**
   - Expose tiering stats via Prometheus metrics
   - hot_count, warm_count, cold_count gauges
   - promotion/demotion counters
   - cold_tier_queries counter

Per CONTEXT.md: Respect isInRamIndex() for query routing.
  </action>
  <verify>./zig/zig build && ./zig/zig build test:unit -- --test-filter "ram_index"</verify>
