  <action>
Create standalone CSV import tool per CONTEXT.md (CLI tool, not built into server):

1. **Create tools/csv_import.zig:**
```zig
const std = @import("std");
const vsr = @import("../src/vsr.zig");

pub fn main() !void {
    var gpa = std.heap.GeneralPurposeAllocator(.{}){};
    defer _ = gpa.deinit();
    const allocator = gpa.allocator();

    const args = try parseArgs(allocator);
    defer args.deinit();

    var client = try connectToArcherDB(args.addresses, allocator);
    defer client.deinit();

    try importCsv(client, args.csv_path, args.options);
}
```

2. **CLI options:**
   - `--addresses` - Comma-separated list of ArcherDB addresses
   - `--csv-path` - Path to CSV file
   - `--batch-size` - Number of rows per batch (default: 1000)
   - `--delimiter` - CSV delimiter (default: comma)
   - `--header` - First row is header (default: true)
   - `--column-mapping` - Map CSV columns to entity fields
   - `--skip-errors` - Continue on parse errors
   - `--dry-run` - Validate without importing

3. **CSV parsing:**
   - Use Zig stdlib for file reading
   - Implement simple CSV parser (handle quotes, escapes)
   - Support customizable column mapping:
     - entity_id column (UUID)
     - latitude column (f64)
     - longitude column (f64)
     - timestamp column (ISO8601 or Unix)
     - metadata columns (JSON)

4. **Batch processing:**
   - Read rows in batches
   - Convert to GeoEvent format
   - Send batch insert requests to ArcherDB
   - Track progress (rows imported, errors, duration)

5. **Error handling:**
   - Report parse errors with line numbers
   - Report connection errors with retry info
   - Summary at end (total rows, errors, duration)

6. **Add to build.zig:**
   - Add csv_import build step
   - Ensure it builds as standalone tool

Per CONTEXT.md: CLI tool only, external tool that connects to ArcherDB.
