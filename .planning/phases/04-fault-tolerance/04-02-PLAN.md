2. Zeroing failed single sectors (allowing repair protocol)
3. Repair from other replicas

1. Create test "FAULT-03: disk read error recovered via cluster repair (R=3)"
   - Create 3-node cluster, commit operations to checkpoint
   - Stop one replica
   - Corrupt grid blocks (simulating disk sectors with read errors)
   - Restart replica
   - Verify: cluster repairs corrupted blocks from healthy replicas
   - Verify: after repair, corrupted areas are no longer faulty (area_faulty returns false)

2. Create test "FAULT-03: multiple sector failures repaired (R=3)"
   - Corrupt multiple non-adjacent sectors on one replica
   - Restart and let cluster repair
   - Verify: all sectors repaired
   - Verify: cluster continues operating normally

3. Create test "FAULT-03: WAL read error triggers repair (R=3)"
   - Corrupt WAL prepare on one replica (simulating read error)
   - Restart replica
   - Verify: enters recovering_head due to checksum failure
   - Verify: repairs from other replicas
   - Verify: returns to normal status

4. Create test "FAULT-03: disjoint read errors across replicas recoverable"
   - Corrupt different sectors on different replicas (disjoint pattern)
   - Ensure each sector intact on at least one replica
   - Restart all replicas
   - Verify: cluster recovers all data through cross-replica repair

Implementation note: The testing storage corrupt() method zeros sectors, which invalidates Aegis128 checksums and simulates read errors that return bad data.
