Create test_infrastructure/benchmarks/workloads/ with:

1. __init__.py - Export all workload classes

2. throughput.py - ThroughputWorkload class:
   - __init__(host: str, port: int, data_config: DatasetConfig)
   - setup() -> None
     - Pre-generate events using data_generator.generate_events()
     - Store for batch insertion
   - execute_one() -> Sample
     - Send batch insert request via HTTP POST to /insert
     - Record latency in nanoseconds
     - Return Sample(latency_ns, timestamp_ns, success)
   - get_events_per_batch() -> int
     - Return batch size for throughput calculation
   - cleanup() -> None

   Uses requests library for HTTP. Batch size: 1000 events per insert (configurable).

3. latency_read.py - LatencyReadWorkload class:
   - __init__(host: str, port: int, entity_ids: List[str])
   - setup() -> None
     - Entity IDs must be pre-inserted
   - execute_one() -> Sample
     - Query by UUID: GET /query/uuid/{entity_id}
     - Record latency in nanoseconds
     - Return Sample
   - cleanup() -> None

4. latency_write.py - LatencyWriteWorkload class:
   - __init__(host: str, port: int, data_config: DatasetConfig)
   - setup() -> None
     - Generate events for single inserts
   - execute_one() -> Sample
     - Single event insert: POST /insert with 1 event
     - Record latency in nanoseconds
     - Return Sample
   - cleanup() -> None

All workloads use time.perf_counter_ns() for timing. All use requests library with timeout=30.

1. orchestrator.py - BenchmarkOrchestrator class:
   - __init__(output_dir: str = "reports/benchmarks")
   - run_throughput_benchmark(topology: int, config: BenchmarkConfig) -> Dict
     - Start fresh cluster with ClusterConfig(node_count=topology)
     - Wait for ready and leader
     - Run warmup using warmup_loader iteration counts
     - Run benchmark using BenchmarkExecutor
     - Calculate events/sec = total_events / duration_sec
     - Return result dict with throughput, percentiles, config
   - run_latency_read_benchmark(topology: int, config: BenchmarkConfig) -> Dict
     - Start cluster, pre-load data
     - Run warmup
     - Run latency benchmark
     - Return result dict with percentiles (P50/P95/P99)
   - run_latency_write_benchmark(topology: int, config: BenchmarkConfig) -> Dict
     - Start cluster
     - Run warmup
     - Run single-event insert benchmark
     - Return result dict with percentiles
   - run_full_suite(topologies: List[int] = [1, 3, 5, 6]) -> Dict
     - Run all benchmarks across all topologies
     - Fresh cluster per benchmark (per CONTEXT.md decision)
     - Return combined results

   Helper: run_isolated_benchmark() context manager per RESEARCH.md Pattern 3.
