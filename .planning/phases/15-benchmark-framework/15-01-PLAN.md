Create test_infrastructure/benchmarks/ directory with:

1. __init__.py - Export main classes:
   - BenchmarkConfig from config
   - BenchmarkExecutor from executor
   - BenchmarkResult (dataclass for results)

2. config.py - BenchmarkConfig dataclass with fields:
   - topology: int (1, 3, 5, 6 node count)
   - time_limit_sec: float (default 60.0)
   - op_count_limit: int (default 10_000)
   - min_samples: int (default 1000) - error if fewer
   - target_samples: int (default 10_000)
   - warmup_iterations: int (default from warmup_protocols.json via warmup_loader)
   - target_cv: float (default 0.10) - coefficient of variation threshold
   - max_stability_runs: int (default 10)
   - data_pattern: str ("uniform", "city_concentrated", "hotspot")
   - data_size: int (default 10_000)
   - seed: Optional[int] (default None for random)

3. executor.py - BenchmarkExecutor class:
   - __init__(config: BenchmarkConfig)
   - run(workload: Callable[[], Sample]) -> BenchmarkResult
     - Implements dual termination (time OR count, whichever first)
     - Records start time with time.perf_counter_ns()
     - Collects samples until time_limit_sec OR op_count_limit
     - Errors if fewer than min_samples collected
   - warmup(workload: Callable[[], Sample]) -> bool
     - Runs warmup_iterations from config
     - Then checks CV stability until target_cv reached or max_stability_runs
     - Returns True if stable, False if hit max without stabilizing

   Sample dataclass: latency_ns: int, timestamp_ns: int, success: bool
   BenchmarkResult dataclass: samples: List[Sample], config: BenchmarkConfig, duration_ns: int, warmup_stable: bool

Use time.perf_counter_ns() for nanosecond precision per RESEARCH.md pitfall guidance.
  </action>

  </verify>
  <done>BenchmarkConfig and BenchmarkExecutor classes exist with dual termination logic and warmup stability check</done>

<task type="auto">
  <name>Task 2: Create statistical analysis and HDR histogram modules</name>
  <files>
    test_infrastructure/benchmarks/stats.py
    test_infrastructure/benchmarks/histogram.py
  </files>
  <action>
1. stats.py - Statistical analysis functions:
   - confidence_interval(samples: List[float], confidence: float = 0.95) -> Tuple[float, float]
     - Uses scipy.stats.t.interval() per RESEARCH.md
     - Returns (low, high) bounds
   - coefficient_of_variation(samples: List[float]) -> float
     - Returns std / mean
   - is_stable(samples: List[float], threshold_cv: float = 0.10) -> bool
     - Returns CV < threshold
   - detect_regression(baseline: List[float], current: List[float], alpha: float = 0.05) -> Tuple[bool, float]
     - Uses scipy.stats.ttest_ind with equal_var=False (Welch's t-test)
     - alternative='greater' for latency (higher = regression)
     - Returns (is_regression, p_value)
   - summarize(samples: List[float]) -> Dict with mean, std, cv, ci_low, ci_high

2. histogram.py - HDR Histogram wrapper:
   - LatencyHistogram class:
     - __init__(min_value: int = 1, max_value: int = 3600_000_000, significant_digits: int = 3)
       - Creates HdrHistogram for microsecond latencies up to 1 hour
     - record(latency_us: int) -> None
     - percentile(p: float) -> int
       - Returns value at percentile (e.g., 50, 95, 99)
     - percentiles() -> Dict[str, int]
       - Returns {"p50": val, "p95": val, "p99": val, "p999": val, "max": val}
     - reset() -> None
     - count() -> int
     - from_samples(samples: List[int]) -> "LatencyHistogram" (class method)
       - Creates histogram and records all samples

Import hdrhistogram library per RESEARCH.md. If import fails, provide fallback using sorted percentile (slower but functional).
  </action>
