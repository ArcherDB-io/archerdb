Create historical data management module and enhance regression detection:

1. Create `benchmarks/history/` directory with `.gitkeep` file
   - This is where JSON files will be stored per CONTEXT.md decision
   - Filename format: YYYYMMDD-HHMMSS-{topology}node-{benchmark_type}.json

2. Create `test_infrastructure/benchmarks/historical.py` with:

   a. BenchmarkMetadata dataclass:
      - timestamp: str (ISO format with Z suffix)
      - git_sha: str (from `git rev-parse HEAD`)
      - git_branch: str (from `git rev-parse --abbrev-ref HEAD`)
      - hostname: str (platform.node())
      - cpu_info: str (platform.processor())
      - memory_gb: float (from /proc/meminfo or psutil)
      - config: dict (benchmark configuration)

   b. capture_metadata(config: dict) -> BenchmarkMetadata:
      - Capture git info via subprocess
      - Capture hardware info
      - Return BenchmarkMetadata instance

   c. save_benchmark_result(history_dir, topology, benchmark_type, results, metadata) -> Path:
      - Generate timestamped filename
      - Combine results + metadata
      - Write JSON with indent=2
      - Return filepath

   d. HistoricalBaseline dataclass:
      - best_value: float
      - best_std: float
      - best_run_timestamp: str
      - best_run_git_sha: str
      - all_values: list[float]

   e. load_best_baseline(history_dir, topology, benchmark_type, metric, higher_is_better) -> Optional[HistoricalBaseline]:
      - Glob all history files for topology/benchmark_type
      - Track best value (max if higher_is_better, min otherwise)
      - Calculate std from all historical values
      - Return HistoricalBaseline with context

   f. get_history_dir() -> Path:
      - Return project_root / "benchmarks" / "history"
      - Create if not exists

3. Update `test_infrastructure/benchmarks/regression.py`:

   a. Add RegressionResult NamedTuple:
      - is_regression: bool
      - trigger: str ("statistical", "absolute", "both", "none")
      - deviation_std: float
      - deviation_pct: float
      - p_value: float

   b. Add detect_regression_dual_threshold() function:
      - Parameters: current_samples, baseline_best, baseline_std, std_threshold=2.0, pct_threshold=10.0, higher_is_better=False
      - Calculate deviation from best (not average)
      - Check statistical threshold (>2 std dev)
      - Check absolute threshold (>10%)
      - Return RegressionResult with trigger type
      - Per CONTEXT.md: "both statistical + absolute thresholds"

   c. Add ci_regression_check(results_path, history_dir, topology) -> int:
      - Load current results from JSON
      - Load best baseline via historical.py
      - Call detect_regression_dual_threshold
      - Return 0 for pass, 1 for regression (exit code for CI)
      - Print clear message on regression

   d. Keep existing compare_to_baseline() for backward compatibility
      - But have it optionally use dual threshold when samples available
  </action>

   a. SDK_LIST constant:
      - ["python", "node", "java", "go", "rust", "zig"]
      - Maps to test paths in tests/sdk_tests/

   b. SDKParityResult dataclass:
      - sdk_name: str
      - throughput_events_per_sec: float
      - p50_ms: float
      - p95_ms: float
      - p99_ms: float
      - success_rate: float
      - error_count: int

   c. run_sdk_benchmark(sdk_name, workload_events, cluster_port, timeout_sec=300) -> SDKParityResult:
      - Generate test data using DatasetConfig with fixed seed for reproducibility
      - Run SDK-specific benchmark script (e.g., tests/sdk_tests/python/benchmark.py)
      - Collect throughput and latency metrics
      - Return SDKParityResult

   d. run_sdk_parity_benchmark(cluster_port, workload_size=10_000) -> Dict[str, SDKParityResult]:
      - Generate identical workload data once using generate_events()
      - Save to temp JSON file for all SDKs to use
      - Run each SDK benchmark sequentially
      - Return results dict keyed by SDK name

   e. verify_sdk_parity(results: Dict[str, SDKParityResult], max_deviation_pct=20.0) -> bool:
      - Calculate average throughput across all SDKs
      - Check each SDK is within max_deviation_pct of average
      - Return True if all SDKs within tolerance, False otherwise
      - Print detailed comparison on failure

   f. SDKParityConfig dataclass:
      - workload_size: int = 10_000
      - operations: List[str] = ["insert", "query_radius", "query_polygon"]
      - timeout_sec: int = 300
      - max_deviation_pct: float = 20.0

2. Create `test_infrastructure/benchmarks/workload_benchmarks.py` with:

   a. WorkloadPatternResult dataclass:
      - pattern: str  # "uniform" or "city_concentrated"
      - topology: int
      - throughput_events_per_sec: float
      - p50_ms: float
      - p95_ms: float
      - p99_ms: float
      - total_events: int
      - duration_sec: float

   b. run_workload_pattern_benchmark(
        pattern: str,
        topology: int,
        cluster: ArcherDBCluster,
        event_count: int = 50_000,
        cities: Optional[List[str]] = None,
      ) -> WorkloadPatternResult:
      - Use DatasetConfig from Phase 11 data generator:
        - pattern="uniform" for BENCH-A-03
        - pattern="city_concentrated" with cities list for BENCH-A-04
      - Generate events using generate_events()
      - Run insert benchmark using BenchmarkExecutor
      - Run query benchmark (radius queries distributed across data region)
      - Collect throughput and latency metrics
      - Return WorkloadPatternResult

   c. DEFAULT_CITIES constant:
      - ["San Francisco", "New York", "Chicago", "Los Angeles", "Seattle", "Austin"]
      - Used for city_concentrated pattern if no cities specified

   d. run_uniform_benchmark(topology, cluster, event_count=50_000) -> WorkloadPatternResult:
      - Convenience wrapper calling run_workload_pattern_benchmark("uniform", ...)
      - Per BENCH-A-03: Run benchmarks with uniform distribution workload

   e. run_city_concentrated_benchmark(
        topology,
        cluster,
        event_count=50_000,
        cities=None,
        concentration=0.8,
      ) -> WorkloadPatternResult:
      - Convenience wrapper for city_concentrated pattern
      - Per BENCH-A-04: Run benchmarks with city-concentrated pattern
      - Uses 80% concentration in cities by default (configurable)

   f. compare_workload_patterns(
        uniform_result: WorkloadPatternResult,
        concentrated_result: WorkloadPatternResult,
      ) -> Dict[str, Any]:
      - Compare throughput and latency between patterns
      - Calculate percentage difference
      - Return comparison dict with metrics and differences
