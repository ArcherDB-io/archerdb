2. Create a startup validation function that tests F_FULLFSYNC on the data directory:

```zig
/// Validates that F_FULLFSYNC is supported on the filesystem.
/// Must be called during IO initialization before any data operations.
/// Fails with actionable error if durability cannot be guaranteed.
pub fn validate_fullfsync_support(data_dir_fd: fd_t) !void {
    if (fullfsync_checked) return;

    // Try F_FULLFSYNC on the data directory fd
    _ = posix.fcntl(data_dir_fd, posix.F.FULLFSYNC, 1) catch |err| {
        log.err("F_FULLFSYNC not supported on this filesystem", .{});
        log.err("ArcherDB requires F_FULLFSYNC for durability guarantees", .{});
        log.err("This typically happens with network filesystems or external drives", .{});
        log.err("Solution: Use a local APFS or HFS+ filesystem", .{});
        return err;
    };

    fullfsync_supported = true;
    fullfsync_checked = true;
    log.info("F_FULLFSYNC support verified - durability guaranteed", .{});
}
```

Current code at lines 810-819 has a commented-out assertion:
```zig
.macos => {
    // TODO: this assert triggers, but the macOS tests on x86_64 work...?
    // assert(b.graph.host.result.cpu.arch == .aarch64);
    return fetch(b, .{
        .url = "https://github.com/archerdb/dependencies/releases/download/" ++
            "18.1.8/llvm-objcopy-aarch64-macos.zip",
        ...
    });
},
```

The issue: x86_64 macOS hosts download aarch64 objcopy binary. This works because Rosetta 2 can run it, but is suboptimal and the assertion was wrong.

Fix by supporting both architectures:
```zig
.macos => {
    const arch = b.graph.host.result.cpu.arch;
    return switch (arch) {
        .aarch64 => fetch(b, .{
            .url = "https://github.com/archerdb/dependencies/releases/download/" ++
                "18.1.8/llvm-objcopy-aarch64-macos.zip",
            .file_name = "llvm-objcopy",
            .hash = "N-V-__8AAFAsVgArdRpU50gjJhqaAUSXsTemKo2A9rCaewUV",
        }),
        .x86_64 => fetch(b, .{
            .url = "https://github.com/archerdb/dependencies/releases/download/" ++
                "18.1.8/llvm-objcopy-x86_64-macos.zip",
            .file_name = "llvm-objcopy",
            // Hash will need to be computed for x86_64 binary
            // If x86_64 binary not available, fall back to aarch64 (Rosetta)
            .hash = "N-V-__8AAFAsVgArdRpU50gjJhqaAUSXsTemKo2A9rCaewUV",
        }),
        else => @panic("unsupported macOS architecture"),
    };
},
```

NOTE: Check if x86_64 macOS objcopy exists at the dependencies URL. If not, the fallback to aarch64 is acceptable (Rosetta handles it). In that case, use a simpler fix:
```zig
.macos => {
    // Both x86_64 and aarch64 macOS use aarch64 binary.
    // x86_64 hosts run it via Rosetta 2 translation.
    return fetch(b, .{
        .url = "https://github.com/archerdb/dependencies/releases/download/" ++
            "18.1.8/llvm-objcopy-aarch64-macos.zip",
        .file_name = "llvm-objcopy",
        .hash = "N-V-__8AAFAsVgArdRpU50gjJhqaAUSXsTemKo2A9rCaewUV",
    });
},
```
