  <action>
Create cluster-specific alerting rules following patterns from archerdb-critical.yaml.

Alert categories:

**Connection Pool Alerts:**
- ArcherDBPoolExhausted (critical): active_connections >= max_connections for >1m
- ArcherDBPoolHighTimeout (warning): acquire_timeout_rate > 10/min for >5m
- ArcherDBPoolHealthCheckFailing (warning): health_check_failed_rate > 5/min for >5m

**Load Shedding Alerts:**
- ArcherDBHighShedRate (warning): shed_requests rate > 100/min for >5m
- ArcherDBCriticalShedRate (critical): shed_requests rate > 1000/min for >2m
- ArcherDBNearShedThreshold (warning): composite_score > 0.7 for >10m (approaching threshold)

**Read Replica Alerts:**
- ArcherDBNoHealthyReplicas (critical): count(healthy_replicas) == 0 for >1m
- ArcherDBHighReplicationLag (warning): replication_lag_ops > 1000 for >5m
- ArcherDBCriticalReplicationLag (critical): replication_lag_ops > 10000 for >2m
- ArcherDBHighFailoverRate (warning): failover_to_leader_rate > 50% of reads for >5m

**Consensus Alerts:**
- ArcherDBFrequentViewChanges (warning): view_changes > 3 in 10m
- ArcherDBQuorumAtRisk (critical): healthy_nodes < phase2_quorum

All alerts include:
- severity label (critical/warning)
- team: database label
- summary, description annotations
- runbook_url pointing to docs.archerdb.io/runbooks/
- remediation annotation with brief fix suggestion

Follow YAML structure from existing rule files.
