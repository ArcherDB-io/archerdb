Integrate LoadShedder into the primary request pipeline and emit shed metrics.

- Add `load_shedder: load_shedding.LoadShedder` to Replica state and initialize with `LoadShedder.initDefault()` in init
- Before enqueueing requests in `ignore_request_message` (primary path), compute load signals:
  - queue_depth = `pipeline.request_queue.count + pipeline.prepare_queue.count`
  - latency_p99_ms = max(read/write) from `metrics.Registry.read_latency.getExtendedStats().p99` and `write_latency` (convert seconds→ms)
  - memory_used_pct = 100 - (available/total * 100) using `connection_pool.getAvailableMemory()` and `getTotalMemory()` fallbacks
- Update shedder signals (`updateQueueDepth`, `updateLatencyP99`, `updateMemoryPressure`) and call `cluster_metrics.updateShedMetrics`
- If `shouldShed()` returns `.shed = true`, call `cluster_metrics.recordShedRequest(retry_after_ms)` and send eviction with reason `overloaded` + retry_after_ms, then return early
- Ensure non-shed requests continue through the normal validation path
  </action>
