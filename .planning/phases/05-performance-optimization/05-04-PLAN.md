<task type="auto">
  <name>Task 1: Execute scaled endurance test with continuous monitoring</name>
  <files>benchmark-results/endurance/</files>
  <action>
Execute sustained load test to validate PERF-08 (system sustains load without degradation).

**Dev server scaling:** Per CONTEXT.md, the dev server (24GB RAM, 8 cores) cannot run a full 24-hour test at production scale. Scale the test to:
- Duration: 4-8 hours (representative sample, validates trends)
- Load: Sustainable rate for lite config (~10-50K events/sec)
- Monitoring: Sample metrics every 5-15 minutes

**Test execution:**

1. Create endurance test directory:
   ```bash
   mkdir -p benchmark-results/endurance/{raw,metrics}
   ```

2. Build with production config:
   ```bash
   ./zig/zig build -Drelease -Dconfig=production
   ```

3. Start cluster for endurance test:
   ```bash
   # Use scripts/dev-cluster.sh if available, or start single node
   ./zig-out/bin/archerdb --data-dir benchmark-results/endurance/data &
   ARCHERDB_PID=$!
   sleep 5
   ```

4. Run endurance test in segments (capture metrics between segments):
   ```bash
   START_TIME=$(date +%s)
   SEGMENT=0

   # Run for 4 hours (14400 seconds) in 30-minute segments
   while [ $(($(date +%s) - START_TIME)) -lt 14400 ]; do
     SEGMENT=$((SEGMENT + 1))
     echo "=== Segment $SEGMENT at $(date) ===" | tee -a benchmark-results/endurance/log.txt

     # Run 30-minute segment
     timeout 1800 ./zig-out/bin/archerdb benchmark \
       --addresses=127.0.0.1:3001 \
       --event-count=1000000 \
       --entity-count=100000 \
       --query-uuid-count=1000 \
       --query-radius-count=100 \
       --clients=10 \
       > benchmark-results/endurance/raw/segment-${SEGMENT}.txt 2>&1 || true

     # Capture system metrics
     echo "$(date +%s),$(ps -o rss= -p $ARCHERDB_PID 2>/dev/null || echo 0)" \
       >> benchmark-results/endurance/metrics/memory.csv

     # Extract throughput from segment
     grep -E "throughput" benchmark-results/endurance/raw/segment-${SEGMENT}.txt \
       >> benchmark-results/endurance/metrics/throughput.csv || true

     # Brief pause between segments
     sleep 10
   done
   ```

5. Cleanup:
   ```bash
   kill $ARCHERDB_PID 2>/dev/null || true
   ```

**Alternative simplified approach (if cluster scripts unavailable):**
Run the full benchmark suite multiple times in sequence and compare results:
```bash
for i in 1 2 3 4 5 6 7 8; do
  echo "=== Run $i at $(date) ===" >> benchmark-results/endurance/log.txt
  ./scripts/run-perf-benchmarks.sh --full --output benchmark-results/endurance/run-$i
  sleep 60
done
```

**Per CONTEXT.md validation criteria:**
