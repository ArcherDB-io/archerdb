<task type="auto">
  <name>Task 2: Generate CPU flame graphs during write-heavy workload</name>
  <files>benchmark-results/baseline/flamegraphs/</files>
  <action>
Generate flame graphs to identify CPU bottlenecks:

1. Ensure FlameGraph tools are available:
   ```bash
   # Check if already installed
   ls tools/FlameGraph/flamegraph.pl 2>/dev/null || \
     git clone https://github.com/brendangregg/FlameGraph.git tools/FlameGraph
   ```

2. Generate flame graph during write benchmark:
   ```bash
   mkdir -p benchmark-results/baseline/flamegraphs
   ./scripts/flamegraph.sh --output benchmark-results/baseline/flamegraphs/write-workload.svg \
     --duration 60 -- ./zig-out/bin/archerdb benchmark --event-count 500000
   ```

3. Generate flame graph during mixed read/write workload:
   ```bash
   ./scripts/flamegraph.sh --output benchmark-results/baseline/flamegraphs/mixed-workload.svg \
     --duration 60 -- ./zig-out/bin/archerdb benchmark \
       --event-count 100000 --query-uuid-count 10000 --query-radius-count 1000
   ```

4. Generate flame graph during spatial query workload:
   ```bash
   ./scripts/flamegraph.sh --output benchmark-results/baseline/flamegraphs/query-workload.svg \
     --duration 60 -- ./zig-out/bin/archerdb benchmark \
       --event-count 10000 --query-radius-count 5000 --query-polygon-count 500
   ```

Interpret flame graphs per RESEARCH.md:
- Wide bars at TOP = hot functions (optimization targets)
- Look for: s2.region_coverer, ram_index, io_uring paths, allocator functions
- Note unexpected time sinks: hash functions, format parsing, etc.
  </action>

- `benchmark-results/baseline/flamegraphs/mixed-workload.svg` exists and is > 1KB
- `benchmark-results/baseline/flamegraphs/query-workload.svg` exists and is > 1KB
  </verify>
  <done>Flame graphs generated for write, mixed, and query workloads showing CPU time distribution</done>
