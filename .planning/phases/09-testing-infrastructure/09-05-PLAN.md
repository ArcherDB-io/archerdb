# Performance Baseline Management

## Overview
Performance baselines are locked reference points used to detect regressions.
Comparisons use explicit thresholds per CONTEXT.md decisions.

## Thresholds
- **Throughput**: Alert if drops more than 5% from baseline
- **Latency P99**: Alert if increases more than 25% from baseline

## Baseline Lifecycle
1. Main branch pushes upload new baseline artifact
2. PRs download baseline and compare
3. Regressions block merge
4. Baselines reset at each release

## Resetting Baselines
If intentional performance changes are made:
1. Delete the `benchmark-baseline` artifact
2. Merge the PR (no comparison on first run)
3. Subsequent PRs compare against new baseline
