<task type="auto">
  <name>Task 3: Create storage alert rules</name>
  <files>observability/prometheus/alerts/storage.yml</files>
  <action>
  Create storage alert rules per user requirements:

  1. Create directory if needed: `mkdir -p observability/prometheus/alerts/`

  2. Create storage.yml with alert rules:
     ```yaml
     groups:
       - name: archerdb_storage
         rules:
           # Performance degradation alerts
           - alert: WriteAmpSpike
             expr: archerdb_compaction_write_amplification > 5
             for: 5m
             labels:
               severity: warning
             annotations:
               summary: "Write amplification spike detected"
               description: "Write amplification is {{ $value }} (threshold: 5)"

           - alert: P99LatencyBreach
             expr: histogram_quantile(0.99, archerdb_query_latency_bucket) > 0.1
             for: 2m
             labels:
               severity: warning
             annotations:
               summary: "P99 query latency exceeded threshold"
               description: "P99 latency is {{ $value }}s (threshold: 100ms)"

           - alert: SpaceAmpThreshold
             expr: archerdb_storage_space_amplification > 3
             for: 10m
             labels:
               severity: warning
             annotations:
               summary: "Space amplification exceeded threshold"
               description: "Space amplification is {{ $value }}x (threshold: 3x)"

           # Critical condition alerts
           - alert: DiskAlmostFull
             expr: (node_filesystem_avail_bytes{mountpoint="/data"} / node_filesystem_size_bytes{mountpoint="/data"}) < 0.05
             for: 5m
             labels:
               severity: critical
             annotations:
               summary: "Disk space critically low"
               description: "Less than 5% disk space remaining"

           - alert: CompactionStall
             expr: rate(archerdb_storage_bytes_written_total[5m]) == 0 and archerdb_compaction_throttle_ratio < 1
             for: 10m
             labels:
               severity: critical
             annotations:
               summary: "Compaction appears stalled"
               description: "No compaction writes for 10 minutes while throttled"

           - alert: CompressionDegraded
             expr: archerdb_compression_ratio < 0.7
             for: 30m
             labels:
               severity: info
             annotations:
               summary: "Compression effectiveness degraded"
               description: "Compression ratio {{ $value }} (expected > 0.7)"

           # Emergency mode
           - alert: EmergencyModeActive
             expr: archerdb_emergency_mode == 1
             labels:
               severity: critical
             annotations:
               summary: "Emergency mode is active"
               description: "ArcherDB entered emergency mode due to critical conditions"
     ```

  3. Use standard Prometheus alerting format
  4. Include severity levels: info, warning, critical
  5. Add meaningful annotations with descriptions
  </action>
