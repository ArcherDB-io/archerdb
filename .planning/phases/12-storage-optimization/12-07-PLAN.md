<objective>
Implement block-level deduplication to reduce storage for repeated values.

Purpose: Geospatial trajectory data often contains repeated locations (e.g., parked vehicles, delivery routes with common stops). Block-level deduplication detects these patterns via content hashing and references existing blocks instead of storing duplicates. This can reduce storage by 10-30% for trajectory-heavy workloads.

Output: DedupIndex with bounded memory, hash-based duplicate detection, integration with table writing.
</objective>

  <action>
  Create src/lsm/dedup.zig with block-level deduplication:

  1. Define DedupConfig:
     ```zig
     pub const DedupConfig = struct {
         enabled: bool = true,
         /// Maximum memory for dedup index (per level)
         index_memory_limit: usize = 64 * 1024 * 1024,  // 64 MiB
         /// Minimum block size to consider for dedup
         min_block_size: usize = 4096,
         /// Use 128-bit hash for collision resistance
         use_128bit_hash: bool = true,
     };
     ```

  2. Define DedupEntry:
     ```zig
     pub const DedupEntry = struct {
         block_address: u64,
         reference_count: u32,
         last_access_tick: u32,  // For LRU eviction
     };
     ```

  3. Define DedupIndex (per-level to bound memory):
     ```zig
     pub const DedupIndex = struct {
         /// Hash -> entry mapping (u64 hash from std.hash.XxHash64)
         entries: std.AutoHashMap(u64, DedupEntry),
         /// Current memory usage estimate
         current_memory: usize,
         /// Config reference
         config: DedupConfig,
         /// LRU tracking tick
         current_tick: u32,
         allocator: std.mem.Allocator,

         pub fn init(allocator: std.mem.Allocator, config: DedupConfig) DedupIndex { ... }
         pub fn deinit(self: *DedupIndex) void { ... }
     };
     ```
     Note: Use u64 hash (XxHash64) by default. Collision probability is negligible.

  4. Implement compute_hash() using Zig stdlib XxHash64:
     ```zig
     const std = @import("std");

     /// Compute content hash for block deduplication
     /// Uses std.hash.XxHash64 from Zig stdlib (built-in, extremely fast)
     pub fn compute_hash(block_content: []const u8) u64 {
         return std.hash.XxHash64.hash(0, block_content);
     }
     ```
     - Use std.hash.XxHash64 from Zig stdlib (no external dependency needed)
     - Collision probability with 64-bit hash is negligible for our use case (1 in 2^64)
     - For optional 128-bit paranoid mode: compute two hashes with different seeds and combine
     - Return u64 hash value

  5. Implement lookup_or_insert():
     ```zig
     pub fn lookup_or_insert(
         self: *DedupIndex,
         block_content: []const u8,
         new_address: u64,
     ) ?u64
     ```
     Logic:
     - Compute hash of block_content
     - If hash exists in entries:
       a. Increment reference_count
       b. Update last_access_tick
       c. Return existing block_address (duplicate found)
     - If hash not found:
       a. Check memory limit
       b. If over limit, evict LRU entries
       c. Insert new entry
       d. Return null (no duplicate)

  6. Implement evict_lru():
     - Remove entries with oldest last_access_tick
     - Continue until under memory limit
     - Track evictions in metrics

  7. Implement decrement_reference():
     - Called when block is deleted/compacted
     - Decrement reference_count
     - Remove entry if count reaches 0

  8. Add unit tests:
     - Test duplicate detection
     - Test memory limit enforcement
     - Test LRU eviction
     - Test reference counting

  Bound dedup index per-level to prevent memory explosion (per research).

  4. Handle dedup block references:

     - archerdb_dedup_index_memory (current index size)

  6. Handle block deletion:
     - When compaction removes a block, call dedup.decrement_reference()
     - This maintains correct reference counts
