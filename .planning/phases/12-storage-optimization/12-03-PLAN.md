<objective>
Integrate block-level compression into the LSM table write and read paths.

Purpose: Apply LZ4 compression to value blocks during compaction to reduce storage footprint by 40-60%. Index blocks remain uncompressed to preserve fast key lookups - this is a deliberate tradeoff per research recommendations.

Output: Compressed value blocks in LSM tables, transparent decompression on read, configuration option to disable compression.
</objective>

<task type="auto">
  <name>Task 2: Integrate compression into table write path</name>
  <files>src/lsm/table.zig</files>
  <action>
  Modify src/lsm/table.zig to compress value blocks during table building:

  1. Import compression module:
     ```zig
     const compression = @import("compression.zig");
     ```

  2. In TableBuilder (or equivalent table writing struct):
     - Add compression_enabled field from config
     - Add compression_buffer: []u8 for temporary compressed output
     - Initialize compression buffer in init() - size = compression.max_compressed_size(block_body_size)

  3. Modify value block finalization:
     - After building value block content, if compression_enabled:
       a. Call compression.compress_block(block_body, compression_buffer)
       b. If compression succeeds (type != .none):
          - Copy compressed data to block body
          - Update block header size
          - Set metadata.compression and metadata.uncompressed_size
       c. If compression doesn't help (type == .none):
          - Keep original uncompressed data
          - Set metadata.compression = .none

  4. Do NOT modify index block building:
     - Index blocks stay uncompressed
     - metadata.compression = .none for index blocks

  5. Track compression statistics:
     - Bytes before compression
     - Bytes after compression
     - Blocks that didn't compress well

  Carefully test this with existing table building tests to ensure no regressions.
  </action>
  <verify>
