Implement adaptive compaction that auto-tunes based on workload patterns.

Purpose: Most deployments shouldn't need manual compaction tuning. Adaptive compaction detects workload type (write-heavy, read-heavy, scan-heavy) and adjusts parameters accordingly. The dual trigger (>20% write throughput change AND >2x space amp) prevents unnecessary adjustments while responding to genuine workload shifts.

Output: AdaptiveState with workload detection, parameter recommendations, integration with forest/compaction.
</objective>

  <name>Task 1: Create adaptive compaction module</name>
  <files>src/lsm/compaction_adaptive.zig</files>
  <action>
  Create src/lsm/compaction_adaptive.zig with workload-aware auto-tuning:

  1. Define WorkloadType:
     ```zig
     pub const WorkloadType = enum {
         write_heavy,   // >70% writes
         read_heavy,    // >70% reads
         balanced,      // 30-70% writes
         scan_heavy,    // >30% range scans
     };
     ```

  2. Define AdaptiveConfig:
     ```zig
     pub const AdaptiveConfig = struct {
         enabled: bool = true,
         /// Write throughput change threshold to trigger evaluation (20%)
         write_throughput_change_threshold: f64 = 0.20,
         /// Space amp threshold for trigger (2x logical)
         space_amp_threshold: f64 = 2.0,
         /// Sliding window duration for workload detection (1 minute)
         window_duration_ms: u64 = 60_000,
         /// Sample interval (1 second)
         sample_interval_ms: u64 = 1000,
         /// Bounds for auto-tuning (guardrails)
         min_compaction_threads: u32 = 1,
         max_compaction_threads: u32 = 4,
         min_l0_trigger: u32 = 2,
         max_l0_trigger: u32 = 20,
     };
     ```

  3. Define AdaptiveState:
     ```zig
     pub const AdaptiveState = struct {
         detected_workload: WorkloadType = .balanced,
         last_writes_per_second: f64 = 0,
         last_reads_per_second: f64 = 0,
         last_scans_per_second: f64 = 0,
         current_space_amp: f64 = 1.0,
         current_write_amp: f64 = 1.0,
         baseline_writes_per_second: f64 = 0,  // For change detection
         last_sample_ns: i128 = 0,
         samples_in_window: u32 = 0,

         // Current tuned parameters
         current_l0_trigger: u32 = 4,
         current_compaction_threads: u32 = 2,
     };
     ```

  4. Implement sample() method:
     - Called every sample_interval_ms
     - Updates rolling statistics (writes/reads/scans per second)
     - Uses EMA (exponential moving average) for smoothing

  5. Implement detect_workload():
     - Calculate write_ratio = writes / (writes + reads)
     - Calculate scan_ratio = scans / (reads + scans)
     - Classify into WorkloadType

  6. Implement should_adapt():
     - Check dual trigger per user decision:
       a. write_throughput_change > threshold (compared to baseline)
       b. space_amp > space_amp_threshold
     - Both must be true to trigger adaptation

  7. Implement recommend_adjustments():
     - Return recommended parameter changes based on workload type:
       - write_heavy: higher L0 trigger, more threads
       - read_heavy: lower L0 trigger, fewer threads
       - scan_heavy: prefer larger runs
       - balanced: moderate settings
     - Apply guardrails (min/max bounds)

  8. Implement update_baseline():
     - Called after adaptation
     - Sets new baseline for change detection

  9. Add unit tests:
     - Test workload classification
     - Test dual trigger logic
     - Test guardrail enforcement
     - Test parameter recommendations
  </action>

  </verify>
