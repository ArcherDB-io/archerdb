<objective>
Implement tiered compaction strategy as an alternative to leveled compaction.

Purpose: Tiered compaction delays merging sorted runs, reducing write amplification by 2-3x for write-heavy workloads at the cost of higher space amplification. This is the preferred strategy for geospatial workloads with frequent location updates.

Output: TieredCompactionConfig, decision logic for tiered compaction, configuration to select strategy, integration with manifest level management.
</objective>

  Create src/lsm/compaction_tiered.zig with tiered compaction logic:

  1. Define CompactionStrategy enum:
     ```zig
     pub const CompactionStrategy = enum {
         leveled,   // Current: aggressive merge, lower space amp
         tiered,    // New: delayed merge, lower write amp
     };
     ```

  2. Define TieredCompactionConfig:
     ```zig
     pub const TieredCompactionConfig = struct {
         /// Size ratio for triggering merge (total_others >= ratio * largest)
         size_ratio: f64 = 2.0,
         /// Minimum sorted runs to merge
         min_merge_width: u32 = 2,
         /// Maximum sorted runs to merge at once
         max_merge_width: u32 = 8,
         /// Trigger compaction if space amp exceeds this (percent)
         max_space_amplification_percent: u32 = 200,
         /// Maximum sorted runs per level before forced compaction
         max_sorted_runs_per_level: u32 = 10,
         /// Prefer partial compaction for better tail latency (per 2025 research)
         /// Partial: worse average throughput, better tail latency at large levels
         /// Full: better average throughput, worse tail latency (write stalls)
         prefer_partial_compaction: bool = true,  // Default true for latency-sensitive geospatial
     };
     ```

  3. Implement should_compact_tiered():
     ```zig
     pub fn should_compact_tiered(
         sorted_run_count: u32,
         total_size: u64,
         largest_run_size: u64,
         logical_size: u64,
         config: TieredCompactionConfig,
     ) bool
     ```
     Returns true if:
     - sorted_run_count >= min_merge_width AND
     - (space_amp > max_space_amplification_percent OR
        size_ratio trigger OR
        sorted_run_count > max_sorted_runs_per_level)

  4. Implement select_compaction_inputs():
     - Select which sorted runs to merge
     - Prefer runs of similar size (within 2x)
     - Limit to max_merge_width runs
     - If prefer_partial_compaction=true: select subset of overlapping runs (better tail latency)
     - If prefer_partial_compaction=false: select all overlapping runs (better average throughput)
     - Return list of run identifiers

  5. Implement estimate_output_size():
     - Sum of input sizes (no deduplication for estimate)
     - Used for resource planning

  6. Add unit tests:
     - Test size ratio trigger
     - Test space amp trigger
     - Test min/max merge width bounds
     - Test run selection prefers similar sizes
     - Test partial vs full compaction selection

  Reference RocksDB Universal Compaction and EDBT 2025 partial compaction patterns from research.
