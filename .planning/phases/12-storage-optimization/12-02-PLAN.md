  <action>
  Create src/lsm/compaction_metrics.zig with write amplification tracking:

  1. Define WriteAmpMetrics struct with atomic counters:
     ```zig
     pub const WriteAmpMetrics = struct {
         // Rolling window counters (total across all levels)
         logical_bytes_written: std.atomic.Value(u64) = std.atomic.Value(u64).init(0),
         physical_bytes_written: std.atomic.Value(u64) = std.atomic.Value(u64).init(0),

         // Per-level tracking for debugging (use constants.lsm_levels)
         level_writes: [constants.lsm_levels]std.atomic.Value(u64),

         // Flush tracking (memtable -> L0)
         flush_bytes: std.atomic.Value(u64) = std.atomic.Value(u64).init(0),
     };
     ```

  2. Implement methods:
     - init(): Initialize all atomic values to 0
     - record_write(level: u8, bytes: u64): Add to physical and per-level counters
     - record_logical_write(bytes: u64): Track application-level writes
     - record_flush(bytes: u64): Track memtable flush bytes
     - write_amplification(): Return physical/logical ratio (1.0 if no logical writes)
     - level_write_amplification(level: u8): Return level_writes[level] / (level > 0 ? level_writes[level-1] : flush_bytes)

  3. Add space amplification calculation:
     ```zig
     pub fn space_amplification(logical_size: u64, physical_size: u64) f64 {
         if (logical_size == 0) return 1.0;
         return @as(f64, @floatFromInt(physical_size)) / @as(f64, @floatFromInt(logical_size));
     }
     ```

  4. Add rolling window support for time-based metrics:
     - RollingWindowMetrics struct with 1min, 5min, 1hr windows
     - Each window: bytes_written, timestamp of window start
     - sample(): Called periodically to rotate windows

  5. Add unit tests:
     - Test write amp calculation accuracy
     - Test per-level tracking
     - Test space amp edge cases (0 logical size)

  Use existing atomic patterns from src/archerdb/metrics.zig.
